# mise 产品介绍

## 什么是 mise？

**mise**（官方标注为 pronounced "meez"，读作 /miːz/）是一个在一个 CLI 中管理开发工具、环境变量与任务的开发环境工具，用项目配置让新 shell、代码检出与 CI 任务获得一致准备好的环境。

它的设计理念来自专业厨房中的 "mise en place"（/miːz ɑːn ˈplɑːs/，意为“准备就绪”）概念，即在开始工作前将所有工具和材料准备妥当，让开发者可以专注于代码本身，而不是环境配置。

---

## 核心优势

### 1. 🔄 统一管理 - 一个工具搞定一切

mise 让你告别同时维护 nvm、pyenv、rbenv、goenv 等多个版本管理工具的烦恼：

- ✅ 支持官方 registry 可验证的 900+ tools
- ✅ 单个配置文件管理所有工具版本
- ✅ 统一的命令行接口
- ✅ 学习一次，掌握所有

```toml
# mise.toml - 单个文件管理所有工具
[tools]
node = "20"
python = "3.12"
rust = "1"
go = "1.22"
kubectl = "1.30"
terraform = "1.8"
```

### 2. ⚡ 直接路径调用 - 像原生一样快

mise 采用革命性的 **PATH 直接修改**机制（不使用 shims 中间层），彻底解决了传统版本管理工具的性能问题：

| 方案 | 每次调用开销 | 体验 |
|-----|------------|------|
| **mise** | 激活后直接使用真实二进制路径；目录切换/提示符刷新时会更新环境 | 感觉不到任何延迟 |
| asdf (bash) | 存在 shim 跳转开销 | 明显感知延迟 |
| asdf (go) | 存在 shim 跳转开销 | 轻微感知延迟 |
| nvm | 存在 shell 函数开销 | 明显感知延迟 |

**技术实现：**
- ❌ 不使用 shims 中间层
- ✅ 直接修改 PATH 环境变量
- ✅ `which node` 返回真实二进制路径
- ✅ 与所有工具 100% 兼容

### 3. 🔐 安全可靠 - 内置供应链安全

mise 重视供应链安全，提供多层防护机制：

- ✅ **信任机制** - 首次使用项目配置需要 `mise trust` 确认，防止恶意代码执行
- ✅ **签名验证** - 支持 GPG、Cosign、SLSA 等签名验证
- ✅ **插件治理** - 积极将社区插件迁移到受控组织
- ✅ **审计日志** - 完整记录工具安装和配置变更

### 4. 🌐 跨平台 - 随处可用

| 操作系统 | 支持状态 | 安装方式 |
|---------|---------|---------|
| **Linux** | ✅ 支持 | apt/dnf/apk/pacman/安装脚本 |
| **macOS** | ✅ 支持 | Homebrew/安装脚本 |
| **Windows** | ✅ 支持 | Scoop/winget/手动安装 |

## 适用场景

### ✅ 多工具项目开发

如果你同时使用多种编程语言进行开发，mise 是你的最佳选择：

**典型场景：**
- 前端项目：Node.js + Python（构建脚本）+ Rust（WASM）
- 后端项目：Go + Python（数据分析）+ Node.js（工具链）
- 全栈项目：Node.js + Python + Ruby + Terraform

```toml
# 全栈项目配置示例
[tools]
node = "20"
python = "3.12"
ruby = "3.3"
terraform = "1.8"
kubectl = "1.30"
```

### ✅ 团队协作 - 统一开发环境

mise 让团队所有成员使用一致的开发环境配置：

1. **版本一致** - 团队通过配置文件约定 Node/Python/Go 等工具版本
2. **配置一致** - 相同的环境变量、任务定义
3. **零配置** - 新成员 clone 代码后只需 `mise install`
4. **无"在我机器上能跑"问题**

```bash
# 新成员上手只需 3 步
git clone https://github.com/your-team/your-project.git
cd your-project
mise install  # 自动安装所有工具并配置环境
```

### ✅ 对性能敏感的开发者

如果你对命令行延迟零容忍：

- mise 每次调用无额外开销
- 目录切换或提示符显示时会调用 hook-env 更新环境，未变更时会尽早返回
- 日常调用工具时不会经过 shim 中间层

### ✅ 关注供应链安全的团队

如果你的团队重视软件供应链安全：

- mise 提供内置签名验证机制
- 支持审计所有工具来源
- 可配置可信软件源白名单

### ✅ CI/CD 环境标准化

在 CI/CD 流水线中使用 mise 确保环境一致性：

```yaml
# GitHub Actions 示例
- name: Setup mise
  uses: jdx/mise-action@v2

- name: Install dependencies
  run: mise install

- name: Run tests
  run: mise run test
```

## 为什么选择 mise？

### vs 传统方案

| 传统方案的问题 | mise 的解决方案 |
|--------------|---------------|
| 常需要多个版本或工具管理器 | 一个 CLI 管理开发工具、环境变量与任务 |
| shims 导致明显延迟 | 直接修改 PATH，无 shim 额外跳转 |
| 无内置环境变量管理 | 内置 env 配置支持 |
| 无任务管理功能 | 内置 task runner |
| Windows 支持有限 | Windows 可通过 Scoop、winget 或手动安装使用 |
| 无供应链安全机制 | 内置信任和签名验证 |

### vs asdf

mise 是 asdf 的现代替代品，保留 `.tool-versions` 兼容并可按需使用 asdf 插件，同时提供显著提升：

- 避免 shim 中间层，`which node` 等命令返回真实二进制路径
- Windows 支持
- 内置供应链安全
- 环境变量和任务管理
- 更友好的用户体验
