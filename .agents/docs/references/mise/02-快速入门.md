# mise 安装与 Shell 激活快速入门

本章节将带你在 5 分钟内完成 mise 的安装、Shell 激活和基础使用。

---

## 第一步：安装 mise

### 方式一：一键安装脚本（推荐）

**Linux / macOS：**
```bash
curl https://mise.run | sh
```

**Windows（PowerShell）：**
```powershell
scoop install mise
```

### 方式二：包管理器安装

> 💡 **提示：** **Homebrew** 是 macOS 系统的包管理器，**Scoop** 和 **winget** 是 Windows 系统的包管理器。

**macOS (Homebrew)：**
```bash
brew install mise
```

**Debian / Ubuntu：**
```bash
sudo add-apt-repository -y ppa:jdxcode/mise
sudo apt update -y
sudo apt install -y mise
```

**Fedora：**
```bash
dnf copr enable jdxcode/mise
dnf install mise
```

**Arch Linux：**
```bash
sudo pacman -S mise
```

**Windows (Scoop)：**
```powershell
scoop install mise
```

**Windows (winget)：**
```powershell
winget install jdx.mise
```

## 第二步：配置 Shell 激活

安装完成后，需要配置 Shell 激活 mise，这样才能自动切换版本。

### Bash
```bash
echo 'eval "$(~/.local/bin/mise activate bash)"' >> ~/.bashrc
```

### Zsh
```bash
echo 'eval "$(~/.local/bin/mise activate zsh)"' >> ~/.zshrc
```

### Fish
```bash
echo '~/.local/bin/mise activate fish | source' >> ~/.config/fish/config.fish
```

### PowerShell
```powershell
(&mise activate pwsh) | Out-String | Invoke-Expression
```

将以上命令添加到 `$PROFILE` 文件中（PowerShell 配置文件）：
```powershell
notepad $PROFILE
```

> 💡 **提示：** 配置完成后需要**重启终端**或重新加载配置文件才能生效。

---

## 第三步：验证安装

重新打开终端后，运行以下命令验证安装：

```bash
# 检查版本
mise --version

# 运行诊断检查
mise doctor
```

如果看到版本信息且 `mise doctor` 没有报错，说明安装成功！

## Hello World 示例

让我们通过一个简单的示例来体验 mise 的核心功能。

### 场景：创建一个 Node.js 项目

#### 1. 创建项目目录
```bash
mkdir my-node-project
cd my-node-project
```

#### 2. 设置 Node.js 版本

使用 `mise use` 命令设置项目使用的 Node.js 版本：

```bash
mise use node@20
```

mise 会自动：
- ✅ 下载并安装当前可解析的 Node.js 20.x 版本
- ✅ 在当前目录创建 `mise.toml` 配置文件
- ✅ 配置当前 Shell 使用 Node.js 20

#### 3. 查看配置文件

此时项目目录下会生成 `mise.toml` 文件：

```toml
[tools]
node = "20"
```

#### 4. 验证版本

```bash
node --version
# 输出：v20.x.x
```

恭喜！你已经成功使用 mise 管理 Node.js 版本了。

## 多工具示例

让我们创建一个使用多种工具的项目：

```bash
# 创建项目目录
mkdir my-fullstack-project
cd my-fullstack-project

# 配置项目使用的工具版本
mise use node@20 python@3.12 go@1.22
```

mise 会自动安装这三个工具，配置文件如下：

```toml
[tools]
node = "20"
python = "3.12"
go = "1.22"
```

验证所有工具版本：

```bash
node --version   # v20.x.x
python --version # Python 3.12.x
go version       # go1.22.x
```

## 体验自动切换功能

mise 最强大的功能之一是**目录自动切换**。让我们来体验一下：

```bash
# 进入项目 A - 使用 Node.js 18
mkdir project-a && cd project-a
mise use node@18
node --version  # v18.x.x

# 进入项目 B - 使用 Node.js 20
mkdir ../project-b && cd ../project-b
mise use node@20
node --version  # v20.x.x

# 回到项目 A - 版本自动切换回 18
cd ../project-a
node --version  # v18.x.x ✨ 自动切换！

# 回到家目录 - 切换回全局版本
cd
node --version  # 系统版本或全局配置版本
```

> ✨ **魔法时刻：** 每次切换目录时，mise 会自动检测并切换到对应项目的工具版本，你完全不需要手动干预。

## 安装项目所有依赖

当你 clone 一个使用 mise 的项目时，只需一行命令即可安装所有工具：

```bash
git clone https://github.com/example/your-project.git
cd your-project
mise install
```

mise 会自动读取 `mise.toml` 配置文件，下载并安装所有需要的工具版本。

---

## 常用命令速查

| 命令 | 说明 |
|-----|------|
| `mise use node@20` | 设置当前项目使用 Node.js 20 |
| `mise use --global node@20` | 设置全局默认版本 |
| `mise install` | 安装当前项目所有工具 |
| `mise list` | 列出所有已安装工具 |
| `mise doctor` | 诊断检查 |
| `mise exec -- node -v` | 在 mise 环境中执行命令 |
