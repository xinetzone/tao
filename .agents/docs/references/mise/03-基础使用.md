# mise use / install / exec / shell 基础命令

本章节详细介绍 mise 的核心命令和基础功能。

---

## 版本管理

### 版本规范

mise 支持多种版本指定方式：

| 方式 | 示例 | 说明 |
|-----|------|------|
| **精确版本** | `node = "20.11.0"` | 精确匹配 20.11.0 |
| **模糊版本** | `node = "20"` | 匹配当前可解析的 20.x 版本 |
| **latest** | `node = "latest"` | 使用当前稳定版 |
| **语义化版本** | `node = ">=20 <21"` | 支持语义化版本范围 |

### 设置版本

#### `mise use` - 设置工具版本

这是最常用的命令，一步完成安装和配置：

```bash
# 设置当前项目使用 Node.js 20
mise use node@20

# 设置全局默认版本
mise use --global node@20

# 同时设置多个工具
mise use node@20 python@3.12 go@1.22

# 使用不同后端安装
mise use npm:@anthropic-ai/claude-code@latest
mise use pipx:black@latest
mise use github:BurntSushi/ripgrep@latest
```

**执行效果：**
1. 下载并安装指定版本（如果未安装）
2. 更新 `mise.toml` 配置文件
3. 立即在当前 Shell 生效

#### asdf 兼容命令

如果你熟悉 asdf，可以继续使用这些命令：

```bash
# 设置本地版本（写入 .tool-versions）
mise local node 20.11.0

# 设置全局版本
mise global node 20.11.0
```

## 安装工具

### `mise install` - 安装工具

```bash
# 安装当前项目配置的所有工具
mise install

# 安装特定工具版本
mise install node@20.11.0

# 强制重新安装
mise install --force node@20

# 从锁文件安装（CI 环境推荐）
mise install --frozen
```

### 查看已安装版本

```bash
# 列出所有已安装工具和版本
mise list

# 列出特定工具的版本
mise list node

# 列出所有可用版本
mise ls-remote node
```

### 卸载工具版本

```bash
# 卸载特定版本
mise uninstall node@18

# 卸载该工具的所有版本
mise uninstall --all node
```

## 环境命令

### `mise exec` - 临时执行命令

在不激活 mise 的情况下，临时使用特定版本执行命令：

```bash
# 使用当前项目版本执行
mise exec -- node my-script.js

# 指定版本运行
mise exec node@18 -- node -v

# 指定多个工具版本
mise exec node@18 python@3.10 -- npm run build
```

### `mise shell` - 临时设置 Shell 环境

为当前 Shell 会话设置工具版本（不修改配置文件）：

```bash
# 当前 Shell 使用 Node.js 18
mise shell node@18

# 验证版本
node --version  # v18.x.x

# 退出 Shell 后恢复正常
exit
```

## 常用命令详解

### 1. `mise doctor` - 诊断检查

检查 mise 安装和配置是否正确：

```bash
mise doctor
```

常见检查项：
- ✅ 安装路径是否在 PATH 中
- ✅ Shell 激活是否正确
- ✅ 配置文件语法
- ✅ 网络连接状态
- ✅ GitHub API 访问

### 2. `mise trust` - 信任配置文件

**安全机制：** 首次进入新项目目录时，mise 不会自动加载配置，需要你确认信任：

```bash
# 信任当前目录的配置
mise trust

# 全局信任特定路径（适合受控环境）
mise settings trusted_config_paths=["/home/user/projects"]
```

关于信任机制的详细安全原理和最佳实践，请查阅 [产品介绍 > 安全可靠](01-产品介绍.md) 和 [最佳实践 > 安全最佳实践](06-最佳实践.md)。

### 3. `mise env` - 查看环境变量

显示 mise 管理的所有环境变量：

```bash
# 查看当前环境变量
mise env

# 查看特定版本的环境变量
mise env node@18
```

### 4. `mise where` - 查看安装路径

显示工具的安装位置：

```bash
# 显示当前版本的安装路径
mise where node

# 显示特定版本的安装路径
mise where node@18
```

### 5. `mise config show` - 显示配置

查看当前生效的配置：

```bash
# 显示合并后的完整配置
mise config show

# 显示当前项目的配置
mise config show --no-global
```

### 6. `mise self-update` - 更新 mise 自身

```bash
# 更新到当前可用版本
mise self-update

# 更新到特定版本
mise self-update v2026.5.13
```

## 环境变量管理

mise 内置支持环境变量管理，在 `mise.toml` 的 `[env]` 节中配置即可。完整的环境变量配置选项（模板语法、条件加载、跨平台等）详见 [配置详解 > 环境变量配置](05-配置详解.md)。

配置后，进入项目目录时这些变量会自动加载：

```bash
echo $NODE_ENV    # development
echo $API_KEY     # your-api-key
```

## 版本锁定

### 生成锁文件

使用 `mise.lock` 锁定精确版本，确保团队和 CI 环境使用锁定后的精确版本：

```bash
# 生成锁文件
mise lock
```

> 💡 **提示：** 需要 mise v2024.1 及以上版本支持锁文件功能。

生成的 `mise.lock` 示例：

```toml
node = "20.11.0"
python = "3.12.3"
go = "1.22.2"
```

### 从锁文件安装

```bash
# 严格按照锁文件安装，不自动升级
mise install --frozen
```

> 💡 **提示：** 使用 `--frozen` 参数需要先生成 `mise.lock` 文件，否则会报错。

> 💡 **最佳实践：** 将 `mise.lock` 提交到 Git 仓库，确保团队所有成员使用锁定后的精确工具版本。
