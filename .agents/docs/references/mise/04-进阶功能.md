# mise Backends / Hooks / Task Runner 进阶功能

本章节介绍 mise 的高级功能，包括多后端架构、钩子机制、任务管理等。

---

## 多后端架构

mise 支持多种后端（Backends）来安装工具，这是它最强大的特性之一。你可以从不同的软件源安装工具，而不仅仅是插件。

### 支持的后端类型

| 后端类型 | 说明 | 示例 |
|---------|------|------|
| **core** | 内置核心工具，官方维护 | `node`, `python`, `go` |
| **asdf** | asdf 插件兼容 | `asdf:node` |
| **aqua** | Aqua Registry | `aqua:ripgrep` |
| **npm** | 直接从 npm 安装 | `npm:prettier` |
| **pipx** | 直接从 PyPI 安装 | `pipx:black` |
| **cargo** | 直接从 crates.io 安装 | `cargo:ripgrep` |
| **github** | GitHub Releases | `github:BurntSushi/ripgrep` |
| **http** | 直接从 URL 下载 | `http:my-tool` |

### 使用示例

```toml
[tools]
# core 后端（默认）
node = "20"
python = "3.12"

# npm 后端 - 直接从 npm 安装 CLI 工具
"npm:@anthropic-ai/claude-code" = "latest"
"npm:prettier" = "latest"

# pipx 后端 - 安装 Python CLI 工具
"pipx:black" = "latest"
"pipx:ruff" = "latest"
"pipx:ansible" = "latest"

# cargo 后端 - 安装 Rust 二进制工具
"cargo:ripgrep" = "latest"
"cargo:bat" = "latest"
"cargo:fd-find" = "latest"

# github 后端 - 从 GitHub Releases 安装
"github:BurntSushi/ripgrep" = "latest"
"github:cli/cli" = "latest"  # GitHub CLI
```

### 命令行使用

```bash
# 使用 npm 后端安装
mise use npm:@anthropic-ai/claude-code@latest

# 使用 pipx 后端
mise use pipx:black@latest

# 使用 github 后端
mise use github:cli/cli@latest
```

## 工具选项

### 安装后钩子（postinstall）

工具安装完成后自动执行命令：

```toml
[tools]
node = { version = "22", postinstall = "corepack enable" }
python = { version = "3.12", postinstall = "pip install --upgrade pip" }
```

### 操作系统限制

指定工具只在特定操作系统上安装：

```toml
[tools]
# 仅在 Linux 和 macOS 安装
ripgrep = { version = "latest", os = ["linux", "macos"] }

# 仅在 Windows 安装
"npm:windows-terminal" = { version = "latest", os = ["windows"] }
```

### 架构限制

进一步限制到特定 CPU 架构：

```toml
[tools]
# 仅在 macOS ARM64 和所有 Linux 安装（跳过 macOS x86_64）
hk = { version = "latest", os = ["linux", "macos/arm64"] }

# 仅在 Linux x86_64 安装
mytool = { version = "latest", os = ["linux/x64"] }
```

**支持的架构标识：**
- `arm64` 或 `aarch64`
- `x64` 或 `x86_64` 或 `amd64`

### 工具依赖

指定工具之间的依赖关系：

```toml
[tools]
python = "3.12.11"
"pipx:ruff" = { version = "latest", depends = ["python"] }
"pipx:black" = { version = "latest", depends = ["python"] }
```

这样，在安装 `ruff` 和 `black` 之前，会确保 `python` 已经安装。

## 任务管理

mise 内置了强大的任务运行器（Task Runner），可以替代 `make`、`npm scripts`、`just` 等工具。

> 💡 任务的基础定义方式详见 [配置详解 > 任务配置](05-配置详解.md)。

### 任务优势

1. **自动安装依赖** - 任务运行前自动安装所有需要的工具
2. **环境隔离** - 任务在正确的 mise 环境中运行
3. **支持依赖** - 可以定义任务之间的依赖关系
4. **多步骤支持** - 支持串行执行多个命令

## 钩子机制

mise 支持多种钩子，可以在特定事件发生时执行命令。

### 支持的钩子类型

| 钩子 | 触发时机 |
|-----|---------|
| `pre_install` | 工具安装前 |
| `post_install` | 工具安装后 |
| `pre_uninstall` | 工具卸载前 |
| `post_uninstall` | 工具卸载后 |
| `pre_run` | 任务运行前 |
| `post_run` | 任务运行后 |
| `pre_activate` | 环境激活前 |
| `post_activate` | 环境激活后 |

### 钩子示例

```toml
[tools]
node = {
  version = "20",
  pre_install = "echo '即将安装 Node.js...'",
  post_install = "corepack enable && npm install -g pnpm"
}

python = {
  version = "3.12",
  post_install = "pip install --upgrade pip setuptools wheel"
}
```

## 别名

### 工具别名

为常用工具配置别名：

```toml
[alias]
n = "node"
p = "python"
g = "go"
```

使用别名：

```bash
mise use n@20    # 等同于 mise use node@20
mise use p@3.12  # 等同于 mise use python@3.12
```

### 命令别名

在 Shell 配置中添加命令别名（推荐）：

```bash
# ~/.bashrc 或 ~/.zshrc
alias m="mise"
alias mi="mise install"
alias mu="mise use"
alias mr="mise run"
alias mx="mise exec --"
```

## 高级环境变量

> 💡 基本环境变量配置（模板语法、条件加载、跨平台等）详见 [配置详解 > 环境变量配置](05-配置详解.md)。

### 条件环境变量

根据操作系统设置不同的环境变量（进阶示例）：

```toml
[env]
PATH = {
  linux = "/usr/local/bin/linux:$PATH",
  macos = "/usr/local/bin/macos:$PATH",
  windows = "C:\\Program Files\\my-tool;%PATH%"
}
```

### 环境变量继承

支持模板语法引用其他变量（进阶用法）：

```toml
[env]
APP_NAME = "my-app"
APP_ENV = "production"
APP_URL = "https://${APP_NAME}.example.com"
FULL_CONFIG = "${APP_ENV}:${APP_URL}"
```

### Secret 管理

使用 mise 的 secret 功能安全存储敏感信息。secrets 会被加密存储，不会明文出现在配置文件中：

```toml
[env]
API_KEY = "{{secret.API_KEY}}"
DATABASE_URL = "{{secret.DATABASE_URL}}"
```

管理 secrets：

```bash
# 设置 secret
mise secret set API_KEY your-secret-key

# 查看 secret
mise secret get API_KEY

# 列出所有 secrets
mise secret list
```

> 💡 **提示：** 需要 mise v2024.1 及以上版本支持 secret 功能。所有 secrets 存储在 `~/.local/share/mise/secrets/` 目录，使用 AES-256-GCM 加密。

## 配置嵌套与继承

mise 支持多层级配置，配置会自动叠加：

```
~/.config/mise/config.toml    # 全局默认
~/work/mise.toml              # 工作区特定配置
~/work/project/mise.toml      # 项目级配置（优先级最高）
```

### 配置合并规则

- 工具版本：子目录覆盖父目录
- 环境变量：合并，子目录优先级高
- 任务：合并，子目录优先级高

### 示例

**全局配置** `~/.config/mise/config.toml`：
```toml
[tools]
node = "18"    # 默认使用 Node.js 18
python = "3.11"

[env]
EDITOR = "code"
```

**项目配置** `~/work/my-app/mise.toml`：
```toml
[tools]
node = "20"    # 覆盖全局配置，使用 Node.js 20

[env]
NODE_ENV = "production"
```

最终效果：
- Node.js: 20（项目配置覆盖）
- Python: 3.11（全局配置）
- EDITOR: code（全局配置）
- NODE_ENV: production（项目配置新增）

## 自动补全

mise 内置支持所有主流 Shell 的自动补全：

```bash
# bash
mise completion bash > /etc/bash_completion.d/mise

# zsh
mise completion zsh > "${fpath[1]}/_mise"

# fish
mise completion fish > ~/.config/fish/completions/mise.fish
```

启用补全后，按 `Tab` 键可以自动补全：
- 命令：`mise in<Tab>` → `mise install`
- 工具名：`mise use nod<Tab>` → `mise use node`
- 版本号：`mise use node@2<Tab>` → 显示所有 20.x 版本
