# mise.toml / config.toml 配置详解

本章节详细介绍 mise 的配置文件格式、全局配置和项目级配置。

---

## 配置文件格式

mise 使用 **TOML**（Tom's Obvious, Minimal Language）作为配置文件格式。TOML 是一种易于阅读的配置文件格式，与 YAML 相比语法更简单，与 INI 相比功能更强大。

### 基本结构

```toml
# 注释以 # 开头

[tools]
# 工具版本配置
node = "20"
python = "3.12"

[env]
# 环境变量配置
NODE_ENV = "production"

[tasks]
# 任务配置
build = "npm run build"
```

---

## 配置文件位置

### 全局配置

**位置：** `~/.config/mise/config.toml`

适用于所有项目的默认配置，通常用于设置个人偏好的工具版本。

```toml
# ~/.config/mise/config.toml
[tools]
node = "20"    # 所有项目默认使用 Node.js 20
python = "3.12"

[settings]
auto_update = true
color = true
```

### 项目配置

**位置：** 项目根目录下的 `mise.toml` 或 `.mise.toml`

仅适用于当前项目的配置。

```toml
# /path/to/project/mise.toml
[tools]
node = "18"    # 该项目特殊使用 Node.js 18

[env]
NODE_ENV = "development"
```

### 遗留格式兼容

mise 兼容 asdf `.tool-versions` 并可按需使用 asdf 插件 的 `.tool-versions` 格式：

```
# .tool-versions
node 20.11.0
python 3.12.3
```

> 💡 **提示：** 你可以使用 `mise use node@20` 将 `.tool-versions` 自动转换为 `mise.toml` 格式。

## `[tools]` 工具配置

### 简单格式

```toml
[tools]
node = "20"
python = "latest"
go = "1.22.2"
```

### 完整格式

当需要配置额外选项时，使用完整格式：

```toml
[tools]
node = {
  version = "20",
  postinstall = "corepack enable",
  os = ["linux", "macos"]
}

python = {
  version = "3.12",
  postinstall = "pip install --upgrade pip",
  depends = []
}
```

### 可用选项

| 选项 | 类型 | 说明 |
|-----|------|------|
| `version` | string | 工具版本号 |
| `postinstall` | string | 安装后执行的命令 |
| `os` | array | 支持的操作系统列表 |
| `depends` | array | 依赖的其他工具 |
| `backend` | string | 指定使用的后端 |

### 后端指定

```toml
[tools]
# 使用 npm 后端
"npm:prettier" = "latest"

# 使用 pipx 后端
"pipx:black" = "latest"

# 使用 cargo 后端
"cargo:ripgrep" = "latest"

# 使用 github 后端
"github:BurntSushi/ripgrep" = "latest"
```

## `[env]` 环境变量配置

### 基本用法

```toml
[env]
NODE_ENV = "production"
API_KEY = "your-api-key"
PYTHONPATH = "./src"
EDITOR = "code"
```

### 模板语法

引用其他环境变量：

```toml
[env]
APP_NAME = "my-app"
APP_ENV = "production"
APP_URL = "https://${APP_NAME}.example.com"
```

### 按操作系统区分

```toml
[env]
PATH = {
  linux = "/opt/linux/bin:$PATH",
  macos = "/opt/macos/bin:$PATH",
  windows = "C:\\opt\\bin;%PATH%"
}
```

### 从 .env 文件加载

```toml
[env]
_.path = ".env"    # 从 .env 文件加载所有变量
```

### 条件加载

通过 `_.if` 配置项可以根据条件决定是否加载环境变量：

```toml
[env]
_.if = { env = "CI" }    # 仅当 CI 环境变量存在且不为空时加载
CI = "true"
```

**支持的条件类型：**
- `env = "VAR_NAME"` - 当指定环境变量存在时
- `os = ["linux", "macos"]` - 当运行在指定操作系统时
- `arch = ["x64", "arm64"]` - 当运行在指定 CPU 架构时

## `[tasks]` 任务配置

### 简单任务

```toml
[tasks]
build = "npm run build"
test = "npm test"
lint = "eslint ."
```

### 多步骤任务

```toml
[tasks]
deploy = [
  "npm run lint",
  "npm run test",
  "npm run build",
  "npm run deploy"
]
```

### 任务完整配置

```toml
[tasks]
build = {
  description = "构建项目",
  run = "npm run build",
  depends = ["lint", "test"],
  env = { NODE_ENV = "production" }
}
```

### 任务依赖

```toml
[tasks]
lint = "eslint ."
test = "jest"
build = { run = "npm run build", depends = ["lint", "test"] }
deploy = { run = "npm run deploy", depends = ["build"] }
```

执行 `mise run deploy` 会按顺序执行：
1. `lint`
2. `test`
3. `build`
4. `deploy`

## `[settings]` 全局设置

```toml
[settings]
# 自动检查 mise 更新
auto_update = true

# 是否显示彩色输出
color = true

# 是否显示进度条
progress = true

# 配置文件搜索深度
config_depth = 3

# 可信配置路径
trusted_config_paths = [
  "/home/user/projects",
  "/home/user/work"
]

# GitHub Token（用于避免 API 限流）
github_token = "ghp_your_token_here"

# 并行安装数量
jobs = 4

# 是否总是保持当前可用版本
always_keep_download = false

# 工具安装根目录
installs_dir = "~/.local/share/mise/installs"
```

### 常用设置说明

| 设置 | 默认值 | 说明 |
|-----|-------|------|
| `auto_update` | `false` | 自动检查并更新 mise |
| `color` | `true` | 启用彩色输出 |
| `progress` | `true` | 显示下载进度条 |
| `jobs` | `4` | 并行安装的工具数量 |
| `github_token` | `null` | GitHub API Token，避免 429 限流 |

## `[plugins]` 插件配置

```toml
[plugins]
# 自定义插件仓库
node = "https://github.com/asdf-vm/asdf-nodejs.git"
python = "https://github.com/asdf-community/asdf-python.git"

# 本地插件
my-plugin = "~/dev/mise-my-plugin"
```

## `[alias]` 别名配置

```toml
[alias]
n = "node"
p = "python"
g = "go"
r = "ruby"
js = "node"
py = "python"
```

使用别名：
```bash
mise use n@20    # 等同于 mise use node@20
mise use py@3.12  # 等同于 mise use python@3.12
```

## 完整配置示例

### 推荐的项目配置

```toml
# mise.toml
[tools]
node = "20"
python = "3.12"
go = "1.22"
rust = "1"
"npm:prettier" = "latest"
"pipx:black" = "latest"
"cargo:ripgrep" = "latest"

[env]
NODE_ENV = "development"
PYTHONPATH = "./src"
EDITOR = "code"

[tasks]
lint = [
  "prettier --check .",
  "black --check .",
  "eslint ."
]

format = [
  "prettier --write .",
  "black ."
]

test = [
  "npm test",
  "pytest"
]

build = {
  description = "构建项目",
  run = "npm run build",
  depends = ["lint", "test"]
}

[settings]
progress = true
jobs = 4
```

## 配置优先级

当存在多个配置文件时，优先级从高到低：

1. **项目本地配置** - `./mise.toml` 或 `./.mise.toml`
2. **父目录配置** - `../mise.toml`（向上搜索，最多 config_depth 层）
3. **全局配置** - `~/.config/mise/config.toml`
4. **环境变量** - `MISE_*` 前缀的环境变量

### 配置合并规则

| 类型 | 合并方式 |
|-----|---------|
| 工具版本 | 子目录覆盖父目录 |
| 环境变量 | 合并，子目录优先级高 |
| 任务 | 合并，子目录优先级高 |
| 设置 | 子目录覆盖父目录 |

## 环境变量配置

除了配置文件，你还可以通过环境变量配置 mise：

| 环境变量 | 说明 |
|---------|------|
| `MISE_CONFIG_DIR` | 配置目录路径 |
| `MISE_DATA_DIR` | 数据目录路径 |
| `MISE_CACHE_DIR` | 缓存目录路径 |
| `MISE_GITHUB_TOKEN` | GitHub API Token |
| `MISE_AUTO_UPDATE` | 自动更新（true/false） |
| `MISE_JOBS` | 并行安装数量 |
| `MISE_COLOR` | 彩色输出（true/false） |

示例：

```bash
# 临时使用不同的配置目录
MISE_CONFIG_DIR=~/.config/mise-alt mise use node@20

# 设置 GitHub Token 避免限流
export MISE_GITHUB_TOKEN="ghp_your_token"
mise install
```

## 配置管理命令

### 查看当前配置

```bash
# 显示合并后的完整配置
mise config show

# 只显示本地配置（不包含全局）
mise config show --no-global

# 显示全局配置
mise config show --global
```

### 编辑配置

```bash
# 编辑全局配置
mise config edit --global

# 编辑本地配置
mise config edit
```

### 定位配置文件

```bash
# 显示全局配置文件路径
mise config which --global

# 显示本地配置文件路径
mise config which
```
