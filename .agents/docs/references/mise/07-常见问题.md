# mise 常见问题

本章节汇总了使用 mise 时最常见的问题和解决方案。

---

## 安装相关问题

### Q: 安装后提示 "command not found: mise"

**原因：** mise 的安装路径没有加入 PATH 环境变量。

**解决方案：**

**Linux / macOS：**
```bash
# 将 mise 路径添加到 PATH
echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.bashrc
# 或 ~/.zshrc
source ~/.bashrc
```

**检查安装位置：**
```bash
# 查看 mise 安装在哪里
which mise

# 如果上面没有找到，检查这些位置
ls -la ~/.local/bin/mise
ls -la /usr/local/bin/mise
```

### Q: 安装过程中网络超时

**原因：** 网络问题导致无法下载 mise 二进制文件。

**解决方案：**

**方法一：使用代理**
```bash
# 设置代理
export http_proxy=http://proxy-server:port
export https_proxy=http://proxy-server:port

# 重新安装
curl https://mise.run | sh
```

**方法二：手动下载**
```bash
# 从 GitHub Releases 手动下载
wget https://github.com/jdx/mise/releases/download/v2026.5.13/mise-v2026.5.13-linux-x64
chmod +x mise-v2026.5.13-linux-x64
mv mise-v2026.5.13-linux-x64 /usr/local/bin/mise
```

### Q: Windows 安装失败

**解决方案：**

**使用 PowerShell 安装（需要管理员权限）：**
```powershell
# 以管理员身份运行 PowerShell
scoop install mise
```

**或使用 Scoop：**
```powershell
scoop install mise
```

## Shell 激活问题

### Q: 自动目录切换不工作

**原因：** Shell 没有正确激活 mise。

**解决方案：**

**确认激活配置：**
```bash
# 检查配置文件是否包含激活命令
cat ~/.bashrc | grep 'mise activate'
cat ~/.zshrc | grep 'mise activate'
```

**重新激活：**
```bash
# Bash
eval "$(mise activate bash)"

# Zsh
eval "$(mise activate zsh)"

# Fish
mise activate fish | source
```

**运行诊断：**
```bash
mise doctor
```

### Q: `which node` 还是显示系统路径

**原因：** mise 没有正确修改 PATH。

**解决方案：**

**检查 PATH：**
```bash
echo $PATH
# 应该包含类似这样的路径：
# /home/user/.local/share/mise/installs/node/20.11.0/bin
```

**手动激活测试：**
```bash
# 手动激活后检查
eval "$(mise activate bash)"
echo $PATH
which node
```

**检查 Shell 类型：**
```bash
# 确认使用的 Shell
echo $SHELL

# 确保使用了正确的 activate 命令
# 比如：使用 zsh 时不要用 bash 的激活命令
```

## 信任机制问题

### Q: 为什么需要 `mise trust`？

关于信任机制的安全原理，详见 [产品介绍 > 安全可靠](01-产品介绍.md)；最佳实践见 [最佳实践 > 安全最佳实践](06-最佳实践.md)。

**场景示例：**
```bash
git clone https://github.com/some-user/some-project.git
cd some-project
# 此时 mise 不会自动加载配置，需要先确认信任
mise trust
```

### Q: 能否跳过信任确认？

**解决方案：**

**方法一：全局信任特定目录**
```toml
# ~/.config/mise/config.toml
[settings]
trusted_config_paths = [
  "/home/user/projects",
  "/home/user/work"
]
```

**方法二：设置环境变量**
```bash
export MISE_TRUSTED_CONFIG_PATHS="/home/user/projects"
```

> ⚠️ **安全警告：** 不要信任整个文件系统（如 `/`），这会带来安全风险。

## GitHub API 限流问题

### Q: 看到 "GitHub API 429 Too Many Requests" 错误

**原因：** 未认证的 GitHub API 请求限流为 60 次/小时，经常会超出。

**解决方案：**

**配置 GitHub Token：**

**方法一：在 mise 配置中设置**
```toml
# ~/.config/mise/config.toml
[settings]
github_token = "ghp_your_personal_access_token_here"
```

**方法二：使用环境变量**
```bash
# ~/.bashrc 或 ~/.zshrc
export GITHUB_TOKEN="ghp_your_personal_access_token_here"
```

**如何获取 Token：**
1. 访问 https://github.com/settings/tokens
2. 点击 "Generate new token" → "Generate new token (classic)"
3. 勾选 "public_repo" 权限（只需要读取权限）
4. 生成后复制 Token

## 工具安装问题

### Q: 工具安装失败怎么办？

**分步排查：**

**步骤 1：查看详细日志**
```bash
# 使用 --verbose 查看详细安装日志
mise install --verbose node@20
```

**步骤 2：检查网络连接**
```bash
# 测试是否能访问 GitHub
curl -I https://github.com
```

**步骤 3：清除缓存重试**
```bash
# 清除缓存
mise cache clean

# 强制重新安装
mise install --force node@20
```

**步骤 4：检查平台支持**
```
某些工具可能不支持你的操作系统或 CPU 架构。
检查工具的官方文档确认支持情况。
```

### Q: 安装速度很慢

**解决方案：**

**方法一：配置代理**
```bash
export http_proxy=http://proxy-server:port
export https_proxy=http://proxy-server:port
mise install
```

**方法二：使用国内镜像**
```bash
# Node.js 使用国内镜像
export NODE_BUILD_MIRROR=https://npmmirror.com/mirrors/node/
mise install node@20
```

**方法三：并行安装**
```toml
# ~/.config/mise/config.toml
[settings]
jobs = 8  # 增加并行安装数量
```

### Q: 某些工具找不到

**原因：** 你需要的工具可能不在 core 后端中。

**解决方案：**

**使用其他后端：**

```toml
[tools]
# 从 npm 安装
"npm:prettier" = "latest"

# 从 PyPI 安装
"pipx:black" = "latest"

# 从 GitHub Releases 安装
"github:BurntSushi/ripgrep" = "latest"
```

**查看支持的工具列表：**
```bash
# 列出可用的工具
mise ls-remote
```

## 版本切换问题

### Q: 进入目录版本没有自动切换

**排查步骤：**

1. ✅ 确认 Shell 已激活：`eval "$(mise activate bash)"`
2. ✅ 确认目录中有 `mise.toml` 或 `.tool-versions`
3. ✅ 确认已经信任该配置：`mise trust`
4. ✅ 运行 `mise doctor` 检查

**常见原因：**

```bash
# 原因 1：没有激活 Shell
# 解决：重新激活
eval "$(mise activate bash)"

# 原因 2：没有信任配置
# 解决：信任当前目录
mise trust

# 原因 3：配置文件位置不对
# 解决：确认 mise.toml 在当前目录
ls -la mise.toml
```

### Q: 模糊版本不自动升级

**原因：** `node = "20"` 安装了当前可解析的 20.x 版本，但之后不会自动升级。

**解决方案：**

```bash
# 手动刷新到当前可解析的 20.x
mise install node@20

# 或者使用 latest
mise use node@latest
```

## 性能问题

### Q: mise 会不会很慢？

**答案：** mise 激活后会把工具路径直接加入 PATH，工具调用走真实二进制路径；目录切换或提示符显示时会调用 hook-env 更新环境，未变更时会尽早返回。

| 操作 | 耗时 |
|-----|------|
| 目录切换/提示符刷新 | 会运行 hook-env，未变更时尽早返回 |
| 调用工具 | 真实二进制路径，无 shim 中间层 |

mise 与其他工具的性能对比详情，详见 [产品介绍](01-产品介绍.md) 和 [特性一览](mise-特性一览.md)。

### Q: 目录切换感觉有延迟

**优化建议：**

1. 减少 `config_depth`（默认 3）
2. 避免过多的嵌套配置文件
3. 使用 `mise doctor` 检查性能瓶颈

```toml
# ~/.config/mise/config.toml
[settings]
config_depth = 2  # 减少搜索深度
```

## 兼容性问题

### Q: mise 和 asdf 能否同时使用？

**答案：** 可以，但不推荐。

**注意事项：**
- 两者都会修改 PATH，可能冲突
- 可能导致版本管理混乱
- 建议迁移到 mise 后再移除 asdf PATH 初始化，避免冲突

### Q: mise 能否读取 .tool-versions？

**答案：** 可以。mise 兼容 asdf `.tool-versions` 文件，并可按需使用 asdf 插件；涉及旧插件或团队脚本时建议验证。

```
# mise 可以识别并读取 .tool-versions 文件
node 20.11.0
python 3.12.3
```

**转换为 mise.toml：**
```bash
mise use node@20 python@3.12
```

### Q: mise 和 nvm/pyenv/rbenv 冲突吗？

**建议：** 迁移到 mise 后卸载这些工具，避免冲突。

```bash
# 卸载 nvm
rm -rf ~/.nvm
# 然后从 .bashrc / .zshrc 中删除 nvm 相关配置

# 卸载 pyenv
rm -rf ~/.pyenv
# 然后从 .bashrc / .zshrc 中删除 pyenv 相关配置
```

## 环境变量问题

### Q: 环境变量没有生效

**解决方案：**

```bash
# 查看 mise 管理的环境变量
mise env

# 检查是否真的生效
echo $NODE_ENV
echo $PYTHONPATH
```

**常见原因：**
1. 没有激活 Shell
2. 没有信任配置
3. 环境变量名称拼写错误

### Q: 如何临时覆盖环境变量？

```bash
# 临时覆盖
NODE_ENV=production mise run build

# 或使用子 shell
(NODE_ENV=production; mise run build)
```

## 任务运行问题

### Q: 任务找不到命令

**原因：** 任务运行时的 PATH 可能不同。

**解决方案：**

确保命令使用的工具在 `[tools]` 中配置：

```toml
[tools]
node = "20"

[tasks]
# ✅ 正确：node 在 tools 中已配置
build = "npm run build"

# ❌ 错误：如果 python 没有配置，pip 可能找不到
# install = "pip install -r requirements.txt"
```

### Q: 任务执行失败，如何调试？

```bash
# 查看详细输出
mise run --verbose build

# 检查任务定义
mise config show | grep -A5 tasks
```

## 升级与卸载

### Q: 如何升级 mise？

```bash
# 升级到当前可用版本
mise self-update

# 升级到特定版本
mise self-update v2026.5.13
```

### Q: 如何完全卸载 mise？

```bash
# 1. 删除所有安装的工具
mise uninstall --all

# 2. 删除 mise 二进制
rm ~/.local/bin/mise

# 3. 删除数据目录
rm -rf ~/.local/share/mise

# 4. 删除配置目录
rm -rf ~/.config/mise

# 5. 从 Shell 配置中删除 activate 行
# 编辑 ~/.bashrc 或 ~/.zshrc，删除 'mise activate' 相关行
```

## 代理配置问题

### Q: 如何配置 mise 使用代理？

**解决方案：**

mise 会自动使用系统的代理配置。你可以通过以下方式设置：

**方法一：使用环境变量**

```bash
# Linux / macOS
export http_proxy=http://proxy-server:port
export https_proxy=http://proxy-server:port
export ALL_PROXY=http://proxy-server:port

# 对于需要认证的代理
export http_proxy=http://username:password@proxy-server:port
export https_proxy=http://username:password@proxy-server:port
```

```powershell
# PowerShell
$env:http_proxy="http://proxy-server:port"
$env:https_proxy="http://proxy-server:port"
$env:ALL_PROXY="http://proxy-server:port"
```

**方法二：在配置文件中设置**

```toml
# ~/.config/mise/config.toml
[settings]
http_proxy = "http://proxy-server:port"
https_proxy = "http://proxy-server:port"
```

**方法三：使用 .env 文件**

```toml
# mise.toml
[env]
_.path = ".env"
```

```
# .env
http_proxy=http://proxy-server:port
https_proxy=http://proxy-server:port
```

### Q: 代理配置不生效怎么办？

**排查步骤：**

```bash
# 1. 检查环境变量是否正确设置
echo $http_proxy
echo $https_proxy

# 2. 测试网络连接
curl -x $http_proxy https://github.com

# 3. 使用 verbose 模式安装，查看详细日志
mise install --verbose node@20
```

## IDE 集成问题

### Q: VS Code 无法识别 mise 管理的工具版本

**原因：** VS Code 的终端可能没有正确激活 mise。

**解决方案：**

**方法一：安装 mise 插件**

1. 在 VS Code 中搜索并安装 "Mise" 插件
2. 重启 VS Code
3. 插件会自动检测项目中的 `mise.toml` 文件

**方法二：配置终端集成**

```json
// .vscode/settings.json
{
  "terminal.integrated.profiles.linux": {
    "bash": {
      "path": "bash",
      "args": ["-l"]
    }
  },
  "terminal.integrated.defaultProfile.linux": "bash"
}
```

**方法三：手动激活**

在 VS Code 终端中手动执行：
```bash
eval "$(mise activate bash)"
```

### Q: JetBrains IDE（IntelliJ、PyCharm 等）如何集成 mise？

**解决方案：**

1. **安装 mise 插件**
   - 在 IDE 的插件市场搜索 "Mise" 并安装

2. **配置 SDK**
   - 打开 `File > Project Structure`
   - 在 `Platform Settings > SDKs` 中点击 `+`
   - 选择相应语言（如 Node.js、Python）
   - 在 `Path` 中选择 mise 安装的版本路径：
     ```
     ~/.local/share/mise/installs/node/20.11.0/bin/node
     ```

3. **自动切换**
   - 安装插件后，IDE 会自动检测 `mise.toml` 文件
   - 打开项目时会自动使用配置的版本

### Q: 如何在 IDE 中使用 mise 管理的 Python 虚拟环境？

**解决方案：**

```bash
# 1. 使用 pipx 安装工具时，mise 会自动创建隔离环境
mise use pipx:black@latest

# 2. 在 PyCharm 中配置解释器
# File > Settings > Project > Python Interpreter
# 点击齿轮图标 > Add > System Interpreter
# 选择路径: ~/.local/share/mise/installs/python/3.12/bin/python
```
