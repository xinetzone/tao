# mise asdf / nvm / pyenv 迁移指南

本章节介绍如何从其他版本管理工具迁移到 mise。

---

## 迁移前准备

在开始迁移之前，先完成 mise 的安装与 Shell 激活：

**安装 mise：**

```bash
curl https://mise.run | sh
```

**激活 mise（Bash / Zsh）：**

```bash
# Bash
echo 'eval "$(~/.local/bin/mise activate bash)"' >> ~/.bashrc

# Zsh
echo 'eval "$(~/.local/bin/mise activate zsh)"' >> ~/.zshrc
```

> 💡 配置完成后需要**重启终端**或重新加载配置文件才能生效。

**验证安装：**

```bash
mise --version
mise doctor
```

## 从 asdf 迁移

mise 兼容 asdf `.tool-versions` 并可按需使用 asdf 插件，迁移非常简单。

### 为什么从 asdf 迁移到 mise？

| 特性 | mise | asdf (bash) |
|-----|------|------------|
| **性能** | ⭐⭐⭐⭐⭐ | ⭐⭐ |
| **Windows 支持** | ✅ 完整支持 | ❌ 不支持 |

完整对比详见 [mise 特性一览](mise-特性一览.md)。

### 迁移步骤

#### 步骤 1：完成迁移前准备
参见上方 [迁移前准备](#迁移前准备) 完成 mise 安装与激活。

#### 步骤 2：自动识别 .tool-versions

mise 可识别项目中的 `.tool-versions` 文件；涉及 asdf 插件、旧命令或团队脚本时仍建议逐项验证：

```bash
cd your-asdf-project
ls -la .tool-versions    # ✅ mise 会自动读取这个文件
```

**验证：**
```bash
# 查看当前生效的版本
mise list

# 验证版本切换
node --version
python --version
```

#### 步骤 3：安装工具版本

```bash
# 安装 .tool-versions 中的所有工具
mise install
```

> 💡 **提示：** mise 不会复用 asdf 已经安装的版本，需要重新安装。但这是一次性操作。

#### 步骤 4（可选）：转换为 mise.toml 格式

如果你想使用 mise 的高级功能（环境变量、任务等），可以转换为 TOML 格式：

```bash
# 一步转换
mise use node@20 python@3.12
```

这会创建 `mise.toml` 文件：

```toml
[tools]
node = "20"
python = "3.12"
```

#### 步骤 5：卸载 asdf（可选）

确认 mise 工作正常后，可以卸载 asdf：

```bash
# 删除 asdf 目录
rm -rf ~/.asdf

# 从 Shell 配置中删除 asdf 相关行
# 编辑 ~/.bashrc 或 ~/.zshrc，删除以下内容：
# . $HOME/.asdf/asdf.sh
# . $HOME/.asdf/completions/asdf.bash
```

### 迁移前后对比

**迁移前（asdf）：**
```
.tool-versions
node 20.11.0
python 3.12.3
```

**迁移后（mise 兼容模式）：**
```
.tool-versions  # ✅ mise 直接读取，无需修改
```

**迁移后（mise 完整功能）：**
```toml
# mise.toml
[tools]
node = "20"
python = "3.12"

[env]
NODE_ENV = "production"

[tasks]
build = "npm run build"
test = "npm test"
```

## 从 nvm 迁移

### 为什么从 nvm 迁移到 mise？

| 特性 | mise | nvm |
|-----|------|-----|
| **支持语言** | 900+ 工具 | 仅 Node.js |
| **性能** | Rust 实现，极快 | Shell 脚本，较慢 |

完整对比详见 [mise 特性一览](mise-特性一览.md)。

### 迁移步骤

#### 步骤 1：完成迁移前准备
参见上方 [迁移前准备](#迁移前准备) 完成 mise 安装与激活。

#### 步骤 2：配置 Node.js 版本

```bash
# 设置全局默认版本
mise use --global node@20

# 设置项目版本
cd your-project
mise use node@18
```

#### 步骤 3：验证

```bash
node --version  # v20.x.x
npm --version
npx --version
```

#### 步骤 4：卸载 nvm（可选）

```bash
# 删除 nvm 目录
rm -rf ~/.nvm

# 从 Shell 配置中删除 nvm 相关行
# 编辑 ~/.bashrc 或 ~/.zshrc，删除：
# export NVM_DIR="$HOME/.nvm"
# [ -s "$NVM_DIR/nvm.sh" ] && \. "$NVM_DIR/nvm.sh"
# [ -s "$NVM_DIR/bash_completion" ] && \. "$NVM_DIR/bash_completion"
```

### nvm 命令对照表

| nvm 命令 | mise 等效命令 |
|---------|--------------|
| `nvm install 20` | `mise install node@20` |
| `nvm use 20` | `mise use node@20` |
| `nvm alias default 20` | `mise use --global node@20` |
| `nvm list` | `mise list node` |
| `nvm ls-remote` | `mise ls-remote node` |
| `nvm uninstall 18` | `mise uninstall node@18` |

## 从多个工具同时迁移

如果你同时使用 nvm + pyenv + rbenv，可以一键迁移：

**迁移前：**
- nvm 管理 Node.js
- pyenv 管理 Python
- rbenv 管理 Ruby

**迁移到 mise：**

```bash
# 1. 完成迁移前准备
参见上方 [迁移前准备](#迁移前准备) 完成 mise 安装与激活。

# 2. 一步配置所有工具
mise use --global node@20 python@3.12 ruby@3.2 go@1.22

# 3. 验证
node --version
python --version
ruby --version
go version

# 4. 卸载旧工具（可选）
rm -rf ~/.nvm
rm -rf ~/.pyenv
rm -rf ~/.rbenv
rm -rf ~/.goenv
```

**效果：**
✅ 一个 CLI 管理开发工具、环境变量与任务
✅ 单个配置文件 `~/.config/mise/config.toml`
✅ 更快的速度
✅ 自动目录切换
✅ 内置环境变量和任务管理

## 迁移注意事项

### 1. 全局 vs 本地版本

**asdf/nvm/pyenv 的习惯：**
```bash
# 设置全局版本
nvm alias default 20
pyenv global 3.12

# 设置本地版本（当前目录）
nvm use 18
pyenv local 3.11
```

**mise 等效命令：**
```bash
# 全局版本
mise use --global node@20
mise use --global python@3.12

# 本地版本（写入 mise.toml）
mise use node@18
mise use python@3.11
```

### 2. 不会自动复用已安装版本

mise 有自己的安装目录，不会使用 asdf/nvm/pyenv 已经安装的版本。这是故意设计的，确保隔离和一致性。

```
# asdf 安装位置
~/.asdf/installs/node/20.11.0

# mise 安装位置
~/.local/share/mise/installs/node/20.11.0
```

> 💡 **提示：** 迁移完成后，可以删除旧工具的安装目录以释放空间。

### 3. 配置文件格式

| 工具 | 配置文件 | 格式 |
|-----|---------|------|
| asdf | `.tool-versions` | 简单文本 |
| nvm | `.nvmrc` | 单行版本号 |
| pyenv | `.python-version` | 单行版本号 |
| mise | `mise.toml` | TOML 格式，功能丰富 |

**从 .nvmrc 迁移：**
```bash
# .nvmrc
18
```

```bash
# 转换为 mise
cd your-project
mise use node@$(cat .nvmrc)
```

**从 .python-version 迁移：**
```bash
cd your-python-project
mise use python@$(cat .python-version)
```

### 4. Shell 钩子对比

| 工具 | 激活方式 | 性能 |
|-----|---------|------|
| mise | 修改 PATH，目录切换时更新 | ✅ 快 |
| asdf | shims 机制，每次调用重定向 | ❌ 每次调用 存在 shim 跳转开销 |
| nvm | Shell 函数，每次 cd 检查 | ❌ 较慢 |

## 回滚方案

如果遇到问题，可以轻松回滚：

```bash
# 1. 卸载 mise
rm ~/.local/bin/mise
rm -rf ~/.local/share/mise
rm -rf ~/.config/mise

# 2. 从 Shell 配置中删除 activate 行
# 编辑 ~/.bashrc 或 ~/.zshrc

# 3. 重新安装原工具（asdf/nvm/pyenv）
# ...
```
