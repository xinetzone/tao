# SSOT vs 镜像页 vs 索引页三层架构

## Type

原则

## Scope

- 适用项目：AgentForge 及任何多入口文档项目
- 适用场景：判断"这段内容应该写在哪里"；识别重复内容的权威源；决定何处放链接何处放正文
- 不适用场景：单页面独立文档；无跨文件引用的场景

## Content

- 记忆内容：
  - **SSOT（真实源）**：内容唯一权威存放处，所有修改只发生在这里
  - **镜像页**：从 SSOT 派生的只读副本或摘要，标注"详见 [SSOT链接]"
  - **索引页**：不含正文内容，仅作为导航入口（toctree / 链接列表）
  - 判断规则：如果两处有相同内容，必须明确哪个是 SSOT，另一个降级为镜像或索引

- 为什么值得记忆：消除了"该在哪里改"的困惑，防止多处修改后不一致
- 它未来能降低什么成本：理解成本（新成员快速定位权威源）、维护成本（只改一处，其他自动/手动同步）
