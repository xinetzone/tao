# 文档维护任务的 5 步通用流程

## Type

原则 + 经验

## Scope

- 适用项目：AgentForge 及任何多入口文档项目
- 适用场景：接收模糊文档维护需求；多文件多入口文档需要同步；识别 SSOT vs 镜像页冲突；迭代结束需要复盘提交
- 不适用场景：单文件简单修改；无多源冲突的文档

## Content

- 记忆内容：
  1. **Discover** — 扫描差异清单（现状 vs 期望，列出所有不一致点）
  2. **Decide** — 结构化用户决策（逐项确认优先级与取舍）
  3. **SoT** — 真实源优先（区分 SSOT 真实源 / 镜像页 / 索引页，只在 SSOT 修改）
  4. **Codify + Automate** — 立制规则 + 脚本 + CI（将决策固化为可执行约束）
  5. **Close** — 复盘 → 归档 → 原子提交（按语义分组提交，归档复盘报告）

- 为什么值得记忆：已在单日 10 轮迭代中验证有效，形成可复用的文档维护闭环
- 它未来能降低什么成本：规划成本（直接套用 5 步框架）、决策成本（SoT 区分消除歧义）、提交质量成本（原子提交 + CI 验证）
