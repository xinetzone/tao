## 第 4 章 关键决策

| 决策点 | 选择 | 备选方案 | 选择依据 |
|--------|------|---------|---------|
| 提取工具 | pdfplumber | pdfminer.six / PyMuPDF | 系统已安装、API 简洁、支持文本+表格 |
| 扫描页处理 | 转为图片嵌入 | OCR 识别 | 避免引入 PaddleOCR 重依赖，保持无损 |
| 章节编号体系 | 帛书本顺序编号 | 通行本编号 | 尊重帛书原著结构，括号标注对应关系 |
| 脚本位置 | `.temp/` | `.agents/scripts/` | 一次性工具不应成为项目资产 |
| 封面渲染 | pypdfium2 | pdf2image | 无需 poppler 外部依赖 |
| 原文标记 | `>` 引用块 | 独立代码块 | 与项目 philosophy/ 文档风格一致 |

### 事后评估

所有决策合理。pdfplumber 的 FontBBox 警告未影响提取质量；帛书本顺序编号使目录结构忠实反映原著逻辑。

## 第 5 章 问题解决

### 问题 1：PDF 分页截断标题（中等）

| 维度 | 详情 |
|------|------|
| 发现时机 | QA 验证阶段 |
| 影响范围 | 19/90 个章节文件 |
| 根因 | PDF 排版换页将章节标题截断为两行，提取时产生孤立碎片 |

## 第 9 章 经验与方法论

### 成功要素

1. **先调研后动手**：Research 阶段确认了 PDF 特征（混合内容、字体问题）和环境能力，避免了盲目试错
2. **中间协议层**：JSON 元数据文件解耦了提取与转换，使两阶段独立可调试
3. **QA 闭环**：验证不是走过场，19 个碎片文件被捕获并修复

### 可复用方法论

**PDF 转结构化文档的三阶段流水线**：

```
提取（逐页文本 + 页面类型判别）
  → 结构分析（章节边界 + 层级关系 → JSON 元数据）
    → 转换（读取元数据 + 格式化输出 + 导航生成）
```

**关键设计模式**：
- 页面类型判别（文本页 vs 扫描页）决定处理策略
- 跨行标题合并处理 PDF 排版换行
- 目录页过滤避免假阳性章节识别
- 帛书本特殊次序的双编号体系（自身序号 + 通行本对照）

### 工具选型经验

- **pdfplumber** 适合中文学术/文学类 PDF，文本提取质量好，但需注意字体警告
- **pypdfium2** 是轻量级页面渲染替代方案（无需 poppler）
- 对于帛书这类古籍注读，章节结构识别需要领域知识（德经/道经次序、中文数字等）
