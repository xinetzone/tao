# 浏览器代理调度规则 (Browser Agent Guidelines)

本文档定义调度浏览器代理（Browser subagent）时的标准化指令规范，避免因措辞模糊导致代理拒绝执行或产物落地错误。

## 1. 截图输出路径

- 所有截图必须保存到项目 `.temp/` 目录下。
- 文件名格式：`{task}-{序号}.png`，例如 `zhihu-dev-01.png`、`api-doc-02.png`。
- 调度指令中必须**显式指定**输出路径，禁止依赖代理默认路径（默认会落到工作目录根目录，污染项目结构）。

```
# 正确示例
截图保存路径：.temp/zhihu-dev-01.png

# 错误示例（缺少路径，代理会保存到根目录）
截图保存到本地
```

## 2. 内容提取指令措辞模板

当需要提取公开网页内容时，使用以下标准措辞，**明确说明目标性质和用途**，避免代理以版权顾虑拒绝：

```
请访问以下 URL 并提取页面主要文本内容：
<URL>

说明：
- 目标页面是公开可访问的 [API 文档 / 官方公告 / 开源项目说明]
- 用途是项目内部参考归档，不涉及二次传播
- 内容不涉及付费墙、受保护区域或版权争议内容
- 仅提取文本正文，无需截图
```

适用场景举例：
- 官方 API 文档（如知乎开放平台、GitHub API 文档）
- 开源项目的 README 或 Changelog
- 技术博客的公开文章（无付费限制）

## 3. 反模式提醒

以下指令措辞易导致代理以版权顾虑拒绝，**应避免**：

| 反模式措辞 | 问题原因 | 替代写法 |
|---|---|---|
| "帮我爬取这个页面" | "爬取"触发版权保护判断 | "访问并提取页面主要文本" |
| "把这篇文章的内容复制给我" | "复制"暗示全文搬运 | "提取核心内容用于参考归档" |
| 不说明页面性质直接给 URL | 代理无法判断是否受保护 | 补充说明页面类型和访问权限 |
| "帮我下载这个文档" | "下载"触发安全策略 | "读取并总结文档要点" |

## 4. 与临时产物规范的关系

截图属于任务中间产物，适用 `.agents/rules/documentation.md` § 3 中的临时产物规则：

- 放入 `.temp/`，任务结束后可安全清理。
- 不得将 `.temp/` 内截图作为长期文档的引用目标。
