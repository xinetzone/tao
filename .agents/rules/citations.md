# 子智能体引用策略规范 (Sub-Agent Citation Rules)

本文档规定了子智能体在生成项目持久化文档时的引用与路径策略，防止临时文件路径泄露到长期维护文档中。

## 1. 生效范围

本规范适用于所有由 AI 智能体（含子智能体）生成的、将被持久化保存到项目仓库中的文档，包括但不限于：

- `.agents/docs/` 下的知识库、技术规范、台账等
- `.agents/docs/superpowers/retrospectives/` 下的复盘报告
- `.agents/docs/superpowers/specs/` 下的设计文档
- `docs/` 下的人类可读文档

## 2. 引用优先级策略

文档中的"数据来源"、"参考资料"、"参考来源"等字段，必须按以下优先级选择引用目标：

| 优先级 | 引用目标 | 示例 |
|--------|---------|------|
| 1 (首选) | 官方永久链接 | `https://docs.python.org/zh-cn/3.16/whatsnew/3.15.html` |
| 2 | 项目内相对路径 | `../../.trae/specs/xxx/spec.md` |
| 3 | 纯文本描述 | `已清理（内容已完全同步至项目文档）` |
| ❌ 禁止 | 本地绝对路径 | `file:///c:/Users/xxx/...` |

## 3. 禁止事项

- **禁止**在持久化文档中引用临时文件路径（如 `python315_whatsnew.md`）
- **禁止**写入包含个人用户名的绝对路径（如 `c:/Users/<name>/...`）
- **禁止**引用预计会被清理的中间产物

## 4. 检查清单

生成持久化文档后，必须自检：

- [ ] 数据来源字段使用的是官方 URL、项目相对路径或已清理标记
- [ ] 无 `c:/Users/` 或 `/home/` 等个人路径泄露
- [ ] 无 `.trae/specs/` 下的临时中间文件引用（除非该文件也属于持久化交付物）

## 5. 模板示例

```markdown
> 数据来源：[Python 3.15 新特性 — 官方文档](https://docs.python.org/zh-cn/3.16/whatsnew/3.15.html)
```

```markdown
| 原始抓取文档 | 已清理（内容已完全同步至项目文档） | ✅ |
```
