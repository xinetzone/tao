# 项目全面复盘 Spec

## Why

本项目需要在阶段性收尾时开展系统化复盘，避免仅停留在零散问题记录或主观总结层面。通过对目标、过程、风险、质量、协作与改进方案进行结构化核验，形成可追溯、可落地、可沉淀的完整复盘报告。

## What Changes

- 新增一套面向本项目的全面复盘工作流程，覆盖目标、执行、风险、质量、协作与改进六个维度。
- 通过读取项目现有规格、变更记录、代码结构、文档、日志与任务材料，形成事实依据清单。
- 对照项目立项目标、核心指标与需求说明，核验交付成果完成度并识别未达成环节与成因。
- 梳理项目全周期执行过程，包括进度规划、资源配置、沟通协作链路与里程碑达成情况。
- 汇总项目执行期间出现的风险、突发问题、应对措施及其有效性。
- 对产品、代码、文档等交付成果进行质量核验，识别缺陷与质量问题根因。
- 总结团队协作效率、责任落实情况与跨角色协作堵点。
- 输出完整复盘报告，包含整体成效、核心问题、经验教训、优化方案与后续行动项。

## Impact

- Affected specs: 项目复盘、交付验收、质量核验、风险治理、团队协作改进
- Affected code: 不直接修改业务代码；需要读取项目文档、规格、变更记录、测试配置、代码结构与日志等资料
- Affected docs: 最终复盘报告应按照项目规则归档至 `.agents/docs/superpowers/retrospectives/`

## ADDED Requirements

### Requirement: 项目目标复盘

The system SHALL 对照项目立项时的预设目标、核心指标与需求说明书，逐一核验所有交付成果的完成度，并梳理未达成目标的具体环节与成因。

#### Scenario: 核验目标完成度

- **WHEN** 执行项目目标复盘
- **THEN** 应识别项目可用的目标来源、需求来源、规格来源与交付成果来源
- **AND** 应输出目标与交付成果之间的对应关系
- **AND** 应标注已完成、部分完成、未完成、无法核验的项目
- **AND** 应说明未达成或无法核验的原因

### Requirement: 执行过程复盘

The system SHALL 梳理项目全周期的进度规划、资源配置、沟通协作链路，复盘里程碑节点完成情况，并识别进度延误、资源错配、沟通断层等问题的核心诱因。

#### Scenario: 梳理执行过程

- **WHEN** 执行过程复盘
- **THEN** 应提取项目阶段、关键任务、里程碑与执行产物
- **AND** 应分析计划与实际执行之间的偏差
- **AND** 应归纳造成偏差的核心诱因

### Requirement: 风险与问题复盘

The system SHALL 汇总项目执行期间出现的风险点、突发问题及已采取的应对措施，评估应对方案有效性，并总结可优化的风险预判与处置机制。

#### Scenario: 评估风险应对

- **WHEN** 执行风险与问题复盘
- **THEN** 应形成风险与问题清单
- **AND** 应记录已采取的应对措施
- **AND** 应评估措施有效性
- **AND** 应提出风险预判与处置机制的优化建议

### Requirement: 成果质量复盘

The system SHALL 对项目交付的产品、代码、文档等成果开展质量核验，对照质量标准梳理存在缺陷，并分析质量问题产生的根因。

#### Scenario: 核验交付质量

- **WHEN** 执行成果质量复盘
- **THEN** 应识别代码、文档、配置、脚本、规格等主要交付成果
- **AND** 应结合现有质量标准、测试结果、静态检查结果或可读取证据进行核验
- **AND** 应输出质量缺陷与根因分析

### Requirement: 团队协作复盘

The system SHALL 总结各参与方协作效率、责任落实情况，梳理跨部门或跨角色协作堵点，并提炼提升团队配合效率的可行方案。

#### Scenario: 分析协作链路

- **WHEN** 执行团队协作复盘
- **THEN** 应基于项目材料中可获得的角色、责任、交接、评审、沟通记录进行分析
- **AND** 应区分事实依据充分的结论与低置信度判断
- **AND** 应提出可执行的协作改进方案

### Requirement: 完整复盘报告输出

The system SHALL 输出完整复盘报告，包含项目整体成效总结、核心问题清单、经验教训提炼、同类项目优化改进方案，以及后续整改与能力提升行动项。

#### Scenario: 生成归档报告

- **WHEN** 完成全部复盘分析
- **THEN** 应生成结构完整的 Markdown 复盘报告
- **AND** 报告应覆盖用户指定的六个复盘模块
- **AND** 报告应明确事实依据、低置信度区域、改进建议与行动项
- **AND** 报告应保存至 `.agents/docs/superpowers/retrospectives/`
