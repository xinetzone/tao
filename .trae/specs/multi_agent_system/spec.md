# 免费大模型多智能体系统 - 产品需求文档

## 概述
- **Summary**: 本项目旨在设计并实现一个无需用户配置API KEY的免费大模型多智能体系统,支持主智能体与子智能体协同工作,具备自主技能探索与创建能力,提供简洁直观的用户交互界面。
- **Purpose**: 解决普通用户使用大模型需要配置API KEY的门槛问题,提供一个开箱即用、功能强大的多智能体协作平台。
- **Target Users**: 需要使用AI辅助但不熟悉API配置的普通用户、开发者、研究人员以及需要多智能体协作的团队。

## Goals
- 实现主智能体与多个子智能体的协同工作机制
- 集成至少3种无需API KEY的免费大模型作为推理引擎
- 支持主智能体自主分析任务并动态创建新技能
- 提供基于模板的智能体创建与管理系统
- 开发简洁直观的用户交互界面
- 确保系统安全性、稳定性和高性能
- 设计可扩展的架构,支持新模型和功能模块的集成

## Non-Goals (Out of Scope)
- 不提供商业级SLA保障
- 不实现付费模型集成(除非用户主动要求)
- 不开发移动端原生应用
- 不实现多用户账户系统(V1版本)
- 不提供模型训练或微调功能

## Background & Context
当前大模型应用普遍需要用户自行获取和配置API KEY,这对非技术用户构成了使用门槛。同时,单一智能体在处理复杂任务时存在局限性。本项目基于现有的taolib库架构,利用其已有的认证、配置管理、任务队列等模块,构建一个完整的多智能体系统。

## Functional Requirements
- **FR-1**: 主智能体能够根据任务需求自主决策并调度子智能体
- **FR-2**: 集成至少3种无需API KEY的免费大模型(如Ollama本地模型、HuggingFace免费API等)
- **FR-3**: 实现模型自动选择与负载均衡机制
- **FR-4**: 主智能体能够分析任务需求并自动生成新技能
- **FR-5**: 支持技能的模块化设计,可动态加载与卸载
- **FR-6**: 提供智能体模板系统,支持快速创建专用子智能体
- **FR-7**: 实现智能体生命周期管理(创建、休眠、唤醒、销毁)
- **FR-8**: 开发Web用户界面,支持任务提交、进度可视化和结果展示
- **FR-9**: 实现输入内容安全过滤机制
- **FR-10**: 建立系统资源监控与异常处理机制

## Non-Functional Requirements
- **NFR-1**: 主智能体决策响应时间不超过5秒
- **NFR-2**: 子智能体创建与部署时间不超过10秒
- **NFR-3**: 系统在10个并发任务下保持稳定运行
- **NFR-4**: 代码测试覆盖率不低于80%
- **NFR-5**: 支持Windows、Linux、macOS主流操作系统
- **NFR-6**: 提供完整的技术文档和用户操作手册

## Constraints
- **Technical**: 基于Python 3.14+,使用FastAPI作为Web框架,集成到现有taolib库架构中
- **Business**: 必须使用免费模型,遵守各免费模型的使用规范和限制
- **Dependencies**: 复用taolib现有模块(auth、config_center、task_queue等)

## Assumptions
- 用户环境可以安装和运行Ollama或其他本地大模型
- 网络环境可以访问HuggingFace等免费API服务
- 用户具备基本的计算机操作能力
- 系统运行在具有足够资源的机器上(至少8GB RAM)

## Acceptance Criteria

### AC-1: 主智能体决策与调度
- **Given**: 用户提交一个复杂任务
- **When**: 主智能体接收任务并进行分析
- **Then**: 主智能体在5秒内做出决策,创建或调度合适的子智能体执行任务
- **Verification**: `programmatic`
- **Notes**: 通过单元测试和性能测试验证响应时间

### AC-2: 免费模型集成
- **Given**: 系统已启动
- **When**: 检查可用的推理引擎
- **Then**: 至少有3种无需API KEY的免费模型可用于推理
- **Verification**: `programmatic`
- **Notes**: 验证模型可用性和基本推理功能

### AC-3: 模型负载均衡
- **Given**: 多个推理请求并发到达
- **When**: 系统处理这些请求
- **Then**: 请求被均匀分发到可用模型,单个模型过载时自动切换
- **Verification**: `programmatic`
- **Notes**: 通过并发测试验证负载均衡效果

### AC-4: 自主技能创建
- **Given**: 主智能体遇到需要新技能的任务
- **When**: 主智能体分析任务需求
- **Then**: 主智能体自动生成并验证新技能,技能可被后续任务使用
- **Verification**: `programmatic`
- **Notes**: 通过端到端测试验证技能创建流程

### AC-5: 智能体模板系统
- **Given**: 用户需要创建特定类型的智能体
- **When**: 用户选择或基于模板创建智能体
- **Then**: 新智能体在10秒内创建完成并具备预期能力
- **Verification**: `programmatic`
- **Notes**: 测试不同类型模板的创建流程

### AC-6: 用户界面功能
- **Given**: 用户访问系统Web界面
- **When**: 用户提交任务、查看进度、获取结果
- **Then**: 界面响应流畅,信息展示清晰,操作无需技术背景
- **Verification**: `human-judgment`
- **Notes**: 通过用户体验测试验证

### AC-7: 内容安全过滤
- **Given**: 用户提交包含敏感内容的输入
- **When**: 系统处理输入
- **Then**: 敏感内容被正确识别和过滤,系统给出友好提示
- **Verification**: `programmatic`
- **Notes**: 使用测试用例验证过滤效果

### AC-8: 系统稳定性
- **Given**: 系统在并发任务下运行
- **When**: 出现异常情况(模型超时、网络错误等)
- **Then**: 系统能够自动恢复,任务继续执行或给出清晰的错误信息
- **Verification**: `programmatic`
- **Notes**: 通过故障注入测试验证
