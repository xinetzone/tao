# 8. 设计方向五：上下文最小化

一个常见误区是把所有东西塞进 `AGENTS.md`——完整数据库 schema、完整 API 文档、完整产品需求、完整历史决策、超长编码规范、几十个示例。

这会造成：Agent 启动上下文过长、重要规则被稀释、更新成本高、规则之间容易冲突。

更好的方法是：

```
AGENTS.md           → 只放稳定、关键、高频规则
.agents/context/    → 放低频但重要的背景资料
docs/               → 放人类文档
代码自身             → 作为最终事实来源
```

`AGENTS.md` 不应该代替文档系统，也不应该代替代码搜索。
