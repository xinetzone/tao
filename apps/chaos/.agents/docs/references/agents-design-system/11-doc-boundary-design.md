# 11. 文档边界设计：人机文档物理隔离

这是具有前瞻性的设计原则：

| 受众 | 目录 | 内容 |
|------|------|------|
| 人类 | `docs/tech/` | API 文档、部署指南、构建规范 |
| 人类 | `docs/general/` | 哲学、数学、通用知识 |
| AI | `.agents/docs/` | AI 专属知识库、架构摘要、分析文档 |
| AI | `.agents/rules/` | 高频执行规则、约定 |

这种隔离避免了"文档既要给人看又要给 AI 看"的两难——内容重点、详细程度、组织方式完全不同。

更推荐在 `.agents/context/architecture.md` 里写"架构摘要"，然后链接到完整 docs。不要让 `AGENTS.md` 取代 `docs/`。
