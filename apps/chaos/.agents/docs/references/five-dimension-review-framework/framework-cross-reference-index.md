| 评审模板 | 修改维度排列顺序 | 设计哲学洞察 §一（认知递进链的论证）；使用说明书 §二（维度填写顺序） |
| 评审模板 | 版本号变更 | 反模式对照表（关联文档引用）；本索引 §一（元数据表） |
| 使用说明书 | 更新填写指导 | 设计哲学洞察 §四（适用边界，若指导与哲学原理不一致） |
| 使用说明书 | 版本号变更 | 反模式对照表（关联文档引用）；本索引 §一 |
| 设计哲学洞察 | 增补新的确认项来源说明 | 使用说明书 §二（若新来源暗示了新的填写注意点） |

| 反模式对照表 | 新增反模式条目 | 体系分析报告 §二（若新条目可映射为新的自指涉线索） |

## 五、版本一致性校验清单

每次任意文档发布新版本时，逐项检查以下内容：

| # | 检查项 | 验证方法 |
|---|--------|---------|
| 1 | 本索引 §一 元数据表中的版本号与实际文件一致 | 打开目标文件，确认文件内声明的版本号 |
| 2 | 所有"关联文档"末尾引用指向的文件名仍然正确 | 逐文档检查末尾"关联文档"段落中的文件名 |
| 3 | 模板章节编号未发生漂移 | 对比上一版本模板的目录结构 |
