# 外循环数据采集规范

> **用途**：当五维度评审框架进入外循环（部署到真实团队、产生实际评审）后，按本规范系统性采集数据，为体系的迭代调整提供量化依据。
>
> **状态**：待外循环启动后启用。内循环阶段仅供预览。
>
> **生成日期**：2026-06-11

---

## 一、采集指标总览

| 指标 | 采集方式 | 采集频率 | 用途 |
|------|---------|---------|------|
| 评审耗时 | 评审者在模板末尾记录起止时间 | 每次评审 | 验证"30分钟"假设，指导模板裁剪粒度 |
| 确认项触发频率 | 统计每条确认项的 ⚠️/❌ 标记次数 | 每季度汇总 | 识别僵尸确认项，为定期收割提供数据 |
| 补充项内容分类 | 对补充项文字进行主题归类 | 每季度汇总 | 发现高频遗漏点，决定是否升级为新的确认项 |
| 文档类型分布 | 记录被评审文档的类型标签 | 每季度汇总 | 验证适用度矩阵假设，发现需要场景裁剪的信号 |
| 评审结论分布 | 统计 ✅/⚠️/❌ 结论比例 | 每季度汇总 | 评估模板的区分度——如果全是 ✅ 或全是 ❌，说明粒度需要调整 |

---

## 二、指标详细定义

### 2.1 评审耗时

| 字段 | 说明 |
|------|------|
| **定义** | 评审者从打开模板到提交评审结论的实际耗时（分钟） |
| **采集位置** | 模板末尾的「评审记录」表，新增"评审耗时（分钟）"行 |
| **数据格式** | 整数，单位分钟 |
| **分析维度** | 按评审者经验水平分组；按文档类型分组 |
| **假设验证** | 当前假设 30 分钟，若中位数持续低于 15 或高于 45，需调整模板粒度 |

### 2.2 确认项触发频率

| 字段 | 说明 |
|------|------|
| **定义** | 某条确认项在所有评审中被标记为 ⚠️ 或 ❌ 的次数占比 |
| **采集方式** | 每次评审完成后，将模板中各确认项的评估标记录入汇总表 |
| **统计周期** | 每季度汇总一次 |
| **阈值建议** | 连续两个季度触发率 = 0% 的确认项，标记为"僵尸条目"，提交定期收割评审 |
| **假设验证** | 当前假设确认项触发频率均衡，若出现长尾分布（少数确认项承担绝大多数触发），说明模板覆盖面需要调整 |

### 2.3 补充项内容分类

| 字段 | 说明 |
|------|------|
| **定义** | 评审者在各维度补充区填写的内容，按主题归类 |
| **分类体系** | 初始分类：技术遗漏 / 流程遗漏 / 文档遗漏 / 风险遗漏 / 其他 |
| **采集方式** | 每季度对积累的补充项文本做人工主题归类 |
| **升级规则** | 同一分类下出现 ≥3 次同类补充项，提交"补充项升级为确认项"提案 |
| **假设验证** | 当前假设补充项会被认真填写，若空白率持续 >50%，需调查原因（时间压力 / 模板太长 / 评审者认为没必要） |

### 2.4 文档类型分布

| 字段 | 说明 |
|------|------|
| **定义** | 被评审文档的类型标签 |
| **标签选项** | 重构方案 / 技术选型 / 架构设计 / 新功能设计 / 性能优化 / 其他 |
| **分析维度** | 按类型统计各维度的"不适用"标注频率 |
| **假设验证** | 当前假设五个维度适用于大多数文档，若某类型下某维度"不适用"率 >80%，考虑为该类型提供场景裁剪 |

### 2.5 评审结论分布

| 字段 | 说明 |
|------|------|
| **定义** | 每次评审的最终结论统计 |
| **采集位置** | 模板 §5.4 评审结论 |
| **健康区间** | ✅ 占比 40-70%，⚠️ 占比 20-50%，❌ 占比 5-20% |
| **异常信号** | ✅ 占比 >90% → 模板可能过于宽松；❌ 占比 >50% → 前置质量可能有问题，或模板过严 |

## 四、数据采集流程

```
每次评审完成
    ↓
评审者填写模板末尾的「评审耗时」
    ↓
维护者（或指定角色）提取评审数据，录入 §三汇总表
    ↓
每季度末：
  1. 汇总确认项触发频率 → 识别僵尸条目
  2. 归类补充项内容 → 识别高频遗漏主题
  3. 统计文档类型分布 → 检查适用度假设
  4. 统计评审结论分布 → 检查模板区分度
    ↓
输出季度数据报告 → 驱动体系调整决策
```

## 五、与定期收割的衔接

本规范采集的数据是定期收割机制的直接输入：

| 定期收割动作 | 依赖的数据 | 数据来源 |
|-------------|-----------|---------|
| 将连续两季度未触发的确认项降级 | 确认项触发频率 (§2.2) | §3.2 汇总表 "触发率" 列 |
| 将高频补充项升级为确认项 | 补充项内容分类 (§2.3) | §3.3 汇总表 "升级建议" 列 |
