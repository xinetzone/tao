# 技术方案评审模板

> **适用场景**：技术方案文档、重构提案、架构设计文档、技术选型建议书的深度评审。
>
> **评审框架**：五维度结构化分析——问题诊断 → 方案设计 → 回归完备性 → 收益可信度 → 整体评价与行动建议。
>
> **使用说明**：评审者按五个维度逐项填写。每个维度包含"确认项"（评估文档已覆盖的内容）和"补充项"（评审者认为应补充的内容）。不必追求面面俱到，根据被评审文档的实际范围和重要性自行裁量。

---

## 维度一：问题诊断评估

> **核心问题**：文档对问题的诊断是否精准？深层原因是否被揭示？

### 1.1 问题定位准确性

| 确认项 | 评估 | 依据 |
|--------|------|------|
| 问题的核心症状是否被清晰描述？ | ✅ / ⚠️ / ❌ | |
| 问题是否被追溯到具体的代码位置或配置项？ | ✅ / ⚠️ / ❌ | |
| 是否区分了"直接原因"和"根本原因"？ | ✅ / ⚠️ / ❌ | |
| 当前约束条件（如环境、版本）是否被明确陈述？ | ✅ / ⚠️ / ❌ | |

### 1.2 问题深度

| 确认项 | 评估 | 依据 |
|--------|------|------|
| 是否揭示了问题形成的演变路径（如何从正常变成问题）？ | ✅ / ⚠️ / ❌ | |
