# 概念先行文档后落方法论

## Type

原则

## Scope

- 适用项目：知识密集型项目的文档体系构建
- 适用场景：构建哲学-工程映射文档；探索性知识整理；从零搭建知识体系
- 不适用场景：已有明确规格的 API 文档生成；纯技术参考文档编写

## Content

- 记忆内容：
  - **核心原则**：不预先决定写什么文档，先自由讨论直到洞察成熟，再"落地为文档"
  - **执行模式**：概念探索（自由对话）→ 用户确认成熟度（"落地为文档"）→ 结构化写作 → 建立互链
  - **α 加速验证**：11 篇哲学文档的构建过程本身验证了"α 加速"规律 — 前期慢、后期快，后期每篇复杂度更高但耗时更短
  - **质量特征**：概念先行产生的文档逻辑内聚度高、互引密度大、读者可从任意入口理解全局

- 为什么值得记忆：指导了 11 篇哲学文档的成功构建，形成了自验证闭环
- 它未来能降低什么成本：规划成本（知道何时不必先规划）、质量成本（概念成熟后写作一次到位）
