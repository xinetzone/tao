# N×N 链接网络构建知识图谱

## Type

经验

## Scope

- 适用项目：具有多篇互相关联文档的知识体系
- 适用场景：构建文档间的全互联引用网络；设计知识图谱导航；维护文档延伸阅读
- 不适用场景：线性教程（有明确前后顺序）；独立的 API 参考页

## Content

- 记忆内容：
  - **核心实践**：每篇新文档的"延伸阅读"部分包含指向所有其他文档的链接，形成 N×N 全互联网络
  - **效果**：任意入口可达全局，读者无需按顺序阅读，每篇文档都是知识图谱的合法入口
  - **维护成本**：O(N) — 每新增 1 篇需更新 N 篇已有文档 + 新文档本身链接 N 篇
  - **可扩展性边界**：~20 篇以内手动可控；超过 20 篇建议引入自动化校验脚本（已实现 `check-philosophy-links.py`）
  - **分层后的路径管理**：文档分组到子目录后，使用 `../目标子目录/文件名.md` 跨目录引用

- 为什么值得记忆：在 11 篇文档中验证了完整可行性，并识别了可扩展性边界
- 它未来能降低什么成本：设计成本（直接复用 N×N 模式）、遗漏成本（配合校验脚本零遗漏）

- 哪些变化会使它失效：如果引入自动链接生成工具（消除手动维护需求）
