## 2. 目标背景

### 2.1 初始目标

- 精简根 `README.md`，将不属于首页的 AI 内部说明拆分出去
- 保留 `README.md` 顶部徽章与链接区域不变
- 将 `AGENTS.md` 收缩为 AI 执行契约与任务路由入口
- 将 `.agents/README.md` 升级为 `.agents/` 目录的唯一主说明页
- 消除三份核心文档之间的重复内容

### 2.2 背景动因

随着项目文档逐步演化，根 `README.md` 一度承载了 AI 内部执行逻辑、目录架构细节、技能说明、变更摘要与贡献规则，已偏离“项目首页”的定位。与此同时，`AGENTS.md` 与 `.agents/README.md` 存在高相似度内容，导致阅读路径不清晰，也增加了后续同步维护成本。

### 2.3 约束条件

- 严禁修改根 `README.md` 顶部的徽章与链接区域
- 保持“人类文档”与“AI 认知上下文”分离
- 遵循既有三层文档架构：`README.md` -> `AGENTS.md` -> `.agents/README.md`
- 尽量通过摘要与跳转，而不是多处重复定义相同内容

## 4. 关键决策

### 4.1 决策清单

| 决策点 | 备选方案 | 最终选择 | 依据 |
|--------|----------|----------|------|
| 根 `README.md` 定位 | 保留细节说明 / 改为入口页 | 改为入口页 | 首页应服务人类开发者快速进入项目 |
| `AGENTS.md` 深度 | 保留详细职责 / 收缩为摘要页 | 收缩为摘要页 | 契约页应聚焦规则、边界与路由 |
| `.agents/` 详细说明落点 | 分散在多个文档 / 单点收口 | 收口到 `.agents/README.md` | 建立单一事实来源，减少同步成本 |
| 处理重复内容方式 | 多处弱差异化改写 / 合并归一 | 合并归一 | 从结构上消除重复，优于表面改写 |

### 4.2 事后评估

✅ **正确决策**：

- 先确定角色边界，再做内容删改，避免来回反复
- 将 `.agents/README.md` 明确为“唯一主说明页”，有效解决重复定义
- 把 `README.md` 从承载细节改为承载导航，显著提升首页稳定性
- 保留 `AGENTS.md` 的摘要与跳转功能，保证 AI 协作入口仍然集中可读

## 5. 问题解决

### 5.1 问题总览

| 问题 | 表现 | 解决方案 |
|------|------|----------|
| 根 `README.md` 过重 | 首页混入大量 AI 内部说明和细节 | 收缩为项目入口与导航页 |
| `AGENTS.md` 偏啰嗦 | 承担了过多目录级展开职责 | 中度精简，保留高层契约与路由 |
| `AGENTS.md` 与 `.agents/README.md` 重复 | 两处存在高度相似描述 | 将详细说明统一收口到 `.agents/README.md` |
| 文档角色不够刚性 | 多个页面都在解释同一件事 | 明确三层边界与唯一主说明页 |

### 5.2 经验教训

**可复用方法**：

- 文档重构优先处理“归属”而不是“措辞”
- 若两个页面高相似，应优先考虑合并而不是继续改写
- 首页内容膨胀时，应直接重构为导航页，而不是继续压缩长段落
- 对于多文档系统，必须建立摘要页与主说明页的严格层级

## 9. 经验方法

### 9.1 成功要素

1. **边界先行**：先定义文档角色，再调整内容
2. **细节下沉**：将展开说明下沉到最合适的目录级说明页
3. **单一事实来源**：避免多个页面重复解释同一结构
4. **入口稳定化**：首页只保留高价值导航信息

### 9.2 可复用方法论

**三层文档边界治理模板**：

```markdown
README.md
- 面向人类开发者
- 保留简介、安装、入口导航

AGENTS.md
- 面向 AI 协作
- 保留契约、路由、边界说明

.agents/README.md
- 面向目录理解与细节展开
- 承担结构、职责、映射、资产分布说明
```

### 9.3 最佳实践

- 用“摘要 + 跳转”替代“跨文档重复展开”
- 对文档重构任务，优先检查页面角色冲突
- 对目录级知识，优先落到目录级 `README`
- 对长期沉淀内容，统一归档到 `.agents/docs/superpowers/`
