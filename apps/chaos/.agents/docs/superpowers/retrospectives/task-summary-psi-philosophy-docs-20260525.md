## 八、经验方法论

### 方法论一：概念先行，文档后落

不是"先决定写什么文档"，而是"先自由讨论直到洞察成熟，再固化"。保证每篇文档都有充分的思考深度。

### 方法论二：N×N 链接网络 = 知识图谱

每篇新文档都与所有已有文档建立双向链接。创造真正的"图"而非"列表"。代价 O(N)，收益任意入口可达全局。

### 方法论三：统一的文档结构模板

所有文档遵循相同章节骨架，降低认知成本，让新文档创建可预测。

### 方法论四：α 加速的自我验证

体系的创建过程证明了体系描述的规律——这是最强的自洽性证据。

### 方法论五：语义化分阶段提交

按认知逻辑（而非时间顺序）组织提交，让 git log 本身成为可读的"建设故事"。
