# 🧪 Tests 智能体契约

这是 `tests/` 目录的局部智能体契约，用于补充根目录 [`../AGENTS.md`](../AGENTS.md) 的全局规则。

## 1. 继承关系

- 本目录下所有任务必须优先遵循根目录 [`../AGENTS.md`](../AGENTS.md)。
- 本文件仅定义测试域补充规则，不重复维护全局规则。
- 如本文件与根目录契约发生冲突，以根目录契约为准。

## 2. 规则增量说明

本子世界为 Fragment 扩展（增量补充），无覆盖父世界规则行为：

| 新增规则文件 | 说明 |
|---|---|
| `.agents/rules/testing.md` | 测试域专项规则，补充全局规则中未涉及的测试工程实践 |

## 3. 测试域规则入口

处理 `tests/` 目录下的测试新增、测试修复、测试重构或测试验证任务前，必须先阅读：

- [`./.agents/rules/testing.md`](./.agents/rules/testing.md)

## 4. 目录职责

- `tests/` 用于存放项目测试代码、测试夹具与测试相关样例。
- `tests/.agents/` 仅用于存放测试域 AI 规则与说明，不承载根级技能、工作流或长期知识库。
- 测试代码必须保持路径可移植性，项目内引用应使用相对路径或包结构导入，禁止写入本机绝对路径。
