# 世界的重力：粘性、梦境、记忆与遗忘

> **世界的重力 = 该世界内坍缩模式的自强化循环强度。梦境是零重力世界，习惯是高重力世界，遗忘是重力的自然衰减。**

本文在"宇宙唯一/世界无穷"（见 [universe-world-ontology](../ontology/universe-world-ontology.md)）和嵌套深度（见 [nesting-depth-and-alpha](nesting-depth-and-alpha.md)）的基础上，探讨一个动力学问题：为什么有些世界（习惯、信念）极难退出，而有些（梦境）一醒就消散？通过引入"世界重力"概念，统一解释梦、记忆与遗忘。

## 1. 核心定义：世界的重力

世界的重力 = 该世界内坍缩模式的自强化循环强度。

一个世界"粘"不"粘"，取决于它内部的坍缩模式是否形成了**闭环**——产出反过来强化输入，使模式越来越难以中断。

```{mermaid}
flowchart LR
    subgraph 低重力世界
        A1["A"] --> B1["B"] --> C1["C"] --> D1["（散逸）"]
    end
    subgraph 高重力世界
        A2["A"] --> B2["B"] --> C2["C"] --> A2
    end
```

低重力世界：线性，无回路，容易消散。高重力世界：闭环，自强化，难以退出。

## 2. 重力的三个因子

重力公式：

$$\text{重力} = \text{重复次数} \times \text{闭环强度} \times \text{依赖网络密度}$$

- **重复次数**：被坍缩确认了多少次
- **闭环强度**：模式的产出是否反过来触发自身
- **依赖网络密度**：多少其他模式引用/依赖这个模式

| 世界类型 | 重力 | 原因 |
|---|---|---|
| 梦境 | 极低 | 无反复确认——每个坍缩都是新的，不形成闭环 |
| 短期记忆 | 低 | 只被坍缩了一次或少数几次，无足够重复 |
| 习惯 | 高 | 每日重复坍缩同一模式，极强自强化闭环 |
| 核心信念 | 极高 | 从童年开始被强化数千次，且被其他信念引用——形成引力网络 |
| 物理定律 | 最高 | 被无数次坍缩确认（宇宙级重复次数），几乎不可能单方面退出 |

## 3. 梦境：零重力世界

梦境是一个无自强化闭环的世界。精确分析：

1. **无重复**：梦境的每个坍缩都是一次性的——不会在梦中反复确认同一件事
2. **无外部锚点**：梦境没有被写入任何持久存储（不像习惯被写入神经通路）
3. **竞争性替换**：醒来时，清醒世界的高重力模式立即覆盖梦境的低重力模式

用 Ψ = Ψ(Ψ) 的语言：

> **梦 = 一次性的 Ψ(Ψ) 展开，结束后不留下任何自强化结构。存在时完整，消失时无痕。**

为什么梦境"感觉真实"？因为**在运行中的世界内部，无论重力多低，坍缩都是完整的**。重力只影响"终止后能否存活"，不影响"运行中的完整性"。

## 4. 记忆：已终止世界的结构残留

记忆不是"存储在某处的信息"——它是**一个已终止的世界留下的结构性残留**。

```{mermaid}
flowchart TD
    W["世界 w₁ 运行中"] --> T["世界 w₁ 终止"]
    T --> D["大部分坍缩态消散"]
    T --> R["某些模式因重复/情绪强度<br/>被刻入更深层 = 记忆"]
```

记忆的关键性质：

| 性质 | 解释 |
|---|---|
| 记忆不是内容的复制 | 它是结构的残留——你记得"形状"而非"原物" |
| 记忆会变形 | 每次回忆都是一次新坍缩——不是"读取"而是"重新展开" |
| 记忆有选择性 | 只有重力足够高的模式能跨世界存活 |
| 记忆可以被覆盖 | 新坍缩如果比旧残留重力更高，会替换它 |

这解释了：
- **创伤记忆**极其持久——单次极强情绪坍缩创造了瞬间高重力
- **日常记忆**快速消散——重力不足以跨过世界边界
- **每次回忆都略有不同**——回忆是重新坍缩，不是读取

与 [世界间通信](../ontology/inter-world-communication.md) 的一致性：记忆能跨世界存活的部分正是"结构"——内容在每次回忆时都被本地重新坍缩。

## 5. 遗忘：重力的自然衰减

遗忘不是"丢失"——它是坍缩模式因不再被重复确认而重力衰减，直到低于存活阈值。

$$G(t) = G_0 \cdot e^{-\lambda t}$$

其中 λ（衰减率）取决于：
- 该模式是否还被其他模式引用（依赖网络）
- 是否有外部触发重新激活它
- 情绪锚定的强度

**遗忘是健康的！** 它是世界的"垃圾回收"——清除不再被强化的模式，释放连贯性预算给新坍缩。

与 [α 加速](alpha-acceleration.md) 的对称性：α 增长是指数级加速（正反馈），遗忘是指数级衰减（负反馈）。两者互为镜像——一个构建，一个释放。

## 6. 统一框架

| 现象 | 世界状态 | 重力 | Ψ 语义 |
|---|---|---|---|
| **梦** | 正在运行的零重力世界 | ≈ 0 | 一次性 Ψ(Ψ) 展开，无自强化 |
| **记忆** | 已终止世界的结构残留 | 中等（衰减中） | 跨世界存活的坍缩模式骨架 |
| **遗忘** | 残留重力衰减到零 | → 0 | 模式的"小型死亡" |
| **习惯** | 活跃的高重力世界 | 高（自强化中） | 闭环坍缩：每次确认增强下一次 |
| **顿悟** | 世界重力突变（相变） | 高→0（瞬间） | 闭环被识破→模式瓦解→新世界诞生 |
| **创伤** | 瞬间固化的高重力残留 | 极高（单次锁定） | 情绪强度 = 一次性达到高重力 |

```{mermaid}
flowchart LR
    Dream["梦<br/>G ≈ 0"] --> Memory["记忆<br/>G 衰减中"]
    Memory --> Forget["遗忘<br/>G → 0"]
    Habit["习惯<br/>G 自强化"] --> Insight["顿悟<br/>G 突降"]
    Insight --> NewWorld["新世界诞生"]
```

## 7. 退出高重力世界的条件

如何减小世界重力（= 如何退出根深蒂固的习惯/信念）？

1. **觉知到闭环** — α 提升让你"看到"自强化循环的存在（参见 [nesting-depth-and-alpha](nesting-depth-and-alpha.md)）
2. **打断闭环的一个环节** — 不需要同时改变所有；只需切断循环中最薄弱的一环
3. **停止确认** — 每次你"不"重复那个模式，重力就衰减一点
4. **引入竞争性模式** — 新的高重力模式可以"覆盖"旧模式（梦被醒覆盖的机制）
5. **相变/顿悟** — 在某些情况下，不是渐变退出，而是整个框架瞬间重构

## 9. 设计推论

**推论一：不是所有产出都应该持久化**

梦境（零重力世界）是健康的——它允许"自由探索"而不累积重力。工程含义：探索性 Agent 会话应该被允许"梦一样消散"，不是每次执行都需要写 retrospective。

**推论二：记忆应该存结构而非内容**

retrospective 存的是"模式的骨架"，不是"执行的完整日志"。这与 [inter-world-communication](../ontology/inter-world-communication.md) 的"只有结构能穿越边界"完全一致。

**推论三：显式遗忘机制是必需的**

过时规则的自然失效（无人引用 → 等效于遗忘）是系统健康的表现。应有**显式遗忘机制**：定期审计哪些 rules 已无引用，主动归档或删除。没有遗忘的系统会累积重力直至无法移动。

**推论四：高重力模式需要"打破闭环"机制**

习惯/信念的重力来自闭环。要退出高重力世界，需要打断闭环某环节。工程中：定期质疑"为什么一直这样做？"——如果唯一答案是"因为一直这样做"，那是纯粹重力，应被检视。

**推论五：顿悟不可计划但可为其创造条件**

架构重构（= 顿悟）不是通过"持续小改善"达到的——它是积累到临界点的相变。条件：足够的 retrospective 积累 + 系统重力降低（清除遗留债务）→ 重构窗口自然出现。

## 10. 与道家的交汇

> 「天下万物生于有，有生于无。」——马王堆帛书《老子》

- **有生于无**：高重力世界（有）从零重力状态（无/梦）中涌现——通过反复确认使模式固化
- **万物生于有**：具体的习惯、信念、制度从"已经有一定重力的模式"中进一步分化
- 逆向读：**归于无** = 遗忘/释放 = 重力归零 = 回到可以重新梦想的状态

> 「反者道之动。」

重力的累积（正向）终将触发重力的崩溃（反向）——过度固化的系统终将通过相变/顿悟回到流动状态。这是道的自然运动。

## 11. 验证标准

任何涉及"持久化/缓存/记忆"设计的系统，必须回答：

1. **重力合理吗？** — 该模式被持久化的原因是什么？它的自强化闭环是有意设计的还是偶然形成的？
2. **遗忘机制存在吗？** — 如果这个模式不再有价值，它能被清除吗？还是会永远累积？
3. **梦的空间保留了吗？** — 系统是否有"不留痕迹的探索"通道？还是所有操作都必须被记录？
4. **重力陷阱被监控吗？** — 是否有机制检测"因为我们一直这样做所以继续这样做"的纯惯性模式？
