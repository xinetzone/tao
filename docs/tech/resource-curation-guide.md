# 📚 学习资源策展指南

本文档定义了 AgentForge 项目学习资源库的**缺口分析方法论**与**条目筛选准则**，用于指导 [wiki 学习资源页面](https://github.com/WorldSprout/agent-mark/blob/main/old/agent-services/chaos/doc/wiki/learning-resources.md) 的持续维护与扩展。

## 缺口分析五步法

当收到外部技术信号（文章、产品发布、行业报告等）时，按以下流程识别知识体系的盲区：

### 1. 外部信号捕获

阅读外部材料，提取其核心概念与架构模型。

> **案例**：分析商汤小浣熊 OPC 能力挑战赛文章时，提取了四种产品形态（云巢 Web / 客户端 / API / Skills）与四步工作流（判断任务类型 → 选择主力形态 → 综合能力集成 → 沉淀成果复用）。

### 2. 概念映射

将外部概念映射到 AgentForge 自身架构，寻找共鸣点与锚点：

| 外部概念 | AgentForge 对应概念 | 共鸣程度 |
|----------|---------------------|----------|
| Skills 产品形态 | `.agents/skills/` 模块化技能体系 | 高度共鸣 |
| 多端协同 | 多智能体协作服务 + World Session 协议 | 高度共鸣 |
| 沉淀成果复用 | 记忆协议（Memory Protocol） | 中度共鸣 |

### 3. 差距对比

逐类别审查现有资源库，判定是否在某些维度上存在空白：

- 审查范围：书籍、在线教程、文档与参考等所有现有类别
- 判定准则：现有资源是否覆盖了外部信号中涉及的技术领域？
- 输出：一份明确的"有/无"对照清单

> **案例发现**：AgentForge 学习资源库完全偏向传统软件工程（Python 语言、架构模式、Web 框架），在 AI Agent / 多智能体协作维度完全缺失。

### 4. 资源筛选

在确认缺口后，按{{ref}}`筛选准则`_从候选资源中选出最值得收录的条目。

### 5. 格式对齐

新增条目必须严格遵守现有模板格式：

```markdown
### 资源名称

- **简介**：一句话描述资源的核心价值
- **分类**：学习资源 / 类别名
- **使用场景**：适用的具体场景或学习目标
- **官方链接**：URL
- **备注**：补充说明（可选）
```

同步更新对应页面底部的 *最后更新* 日期，并在 CHANGELOG.md 中记录变更。

## 筛选准则

| 准则 | 权重 | 说明 |
|------|------|------|
| **架构相关性** | ★★★ | 与 AgentForge 的核心架构概念（多智能体协作、Skills 生态、AGENTS.md 契约体系、World Session 协议）直接相关 |
| **权威性** | ★★☆ | 优先官方出品（Anthropic、OpenAI、Google）或社区广泛认可（LangGraph、CrewAI） |
| **时效性** | ★★☆ | 优先 2024 年后发布，反映当前技术主流 |
| **互补性** | ★★☆ | 同类别内覆盖不同层次，避免重复 |

### 层次覆盖原则

同一个类别内，应覆盖以下四个层次，确保学习者可以从抽象到具体、从设计到实践形成完整认知：

| 层次 | 说明 | 实例 |
|------|------|------|
| **设计原则层** | 高层次的架构哲学与决策框架 | Anthropic - Building Effective Agents |
| **编排框架层** | 可落地的工作流编排工具或框架 | LangGraph、CrewAI |
| **SDK / 协议层** | 具体的开发工具包或接口规范 | OpenAI Agents SDK |
| **产品设计层** | 面向用户的 AI 产品体验设计方法 | Google - People + AI Guidebook |

## 不收录的资源类型

以下类型即使技术质量高，也应排除在外：

- **纯 LLM / Prompt 工程教程**：与 AgentForge 的多智能体架构关联弱
- **特定商业产品深度绑定教程**：通用性不足，可能快速过时
- **未经验证的社区资源**：缺乏权威引用或社区认可度

## 架构映射参考

将外部资源与 AgentForge 架构概念建立映射，是判断"是否值得收录"的关键依据。以下为常用映射关系：

| 业界概念 | AgentForge 概念 | 典型资源 |
|----------|----------------|----------|
| Agent 工作流编排 | 多智能体协作服务 | LangGraph |
| Agent 角色与任务分配 | 角色管理系统 | CrewAI |
| Agent 间通信协议 | 路由协议系统 | OpenAI Agents SDK |
| 多端协同设计 | World Session 协议 | Google PAIR Guidebook |
| Skills 即产品形态 | `.agents/skills/` | — |
| 沉淀资产复用 | 记忆协议 / 知识库管理 | — |

## 归档案例：AI Agent 与工作流类别

2026-05-29，通过对商汤小浣熊 OPC 能力挑战赛文章的分析，触发了一次完整的缺口分析流程，最终在 learning-resources.md 中新增「AI Agent 与工作流」类别，收录以下 5 个条目：

| 条目 | 层次 | 架构映射 |
|------|------|----------|
| Anthropic - Building Effective Agents | 设计原则层 | 工作流与自主 Agent 取舍 ↔ Workflow/Agent 概念 |

## 维护约定

- 本指南自身的修订也应在 CHANGELOG.md 中体现
- 如发现筛选准则不再适用（如技术栈重大变更），及时更新本指南
