# 第二章：架构之道 - Sphinx 整体结构

> 视之而弗见，名之曰微。听之而弗闻，名之曰希。捪之而弗得，名之曰夷。三者不可致诘，故混而为一。一者，其上不皦，其下不昧，绳绳呵不可名也，复归于无物。是谓无状之状，无物之象，是谓惚恍。随而不见其后，迎而不见其首。

## 2.1 Sphinx 整体架构

Sphinx 的整体架构，可以用"一核两翼三域"来概括：

**一核** - `Sphinx` 类为核心，统摄全局
**两翼** - 读入与写出为两翼
**三域** - 源文本、内部表示、输出格式为三域

```
        ┌─────────────────┐
        │   Sphinx 应用   │
        └────────┬────────┘
                 │
        ┌────────┴────────┐
        │                 │
┌───────▼───────┐   ┌────▼────────┐
│ 读取与解析    │   │ 构建与输出  │
└───────┬───────┘   └────┬────────┘
        │                │
┌───────▼───────┐   ┌────▼────────┐
│ BuildEnv     │   │  Builder    │
│ (环境)       │   │ (构建器)    │
└───────────────┘   └─────────────┘
```

> 道生一，一生二，二生三，三生万物。

## 2.2 核心组件解析

### 2.2.1 Sphinx 类 - 众妙之门

`Sphinx` 类位于 `sphinx.application` 模块，是整个系统的入口。

```python
class Sphinx:
    def __init__(self, srcdir, confdir, outdir, doctreedir, buildername, ...):
        # 初始化应用
```

它负责：
- 配置管理
- 扩展加载
- 构建环境创建
- 构建流程协调

### 2.2.2 BuildEnvironment - 道之枢纽

`BuildEnvironment` 位于 `sphinx.environment` 模块，是文档处理的核心。

> 谷神不死，是谓玄牝。玄牝之门，是谓天地之根。绵绵呵若存，用之不勤。

环境犹如玄牝之门，是文档转换的根本，绵绵不绝，用之不尽。

环境负责：
- 文档发现与索引
- 交叉引用解析
- 依赖关系管理
- 状态持久化

### 2.2.3 Builder - 器之妙用

`Builder` 位于 `sphinx.builders` 模块，负责将文档树转换为具体输出格式。

> 朴散则为器，圣人用之则为官长，夫大制无割。

不同的 Builder 是不同的器，圣人（开发者）用之则为官长（管理者），但大制无割（大道不割裂）。

常见 Builder：
- `HTMLBuilder` - 生成 HTML
- `LaTeXBuilder` - 生成 LaTeX
- `TextBuilder` - 生成纯文本
- `ManpageBuilder` - 生成手册页
- `EPUBBuilder` - 生成 EPUB

---

**本章要义**：Sphinx 以环境为枢，以构建器为用，构成完整的文档生成之道。
